# プロンプトパターン集

## 概要

よく使うプロンプトパターンのテンプレート集。

---

## パターン1: コンテンツ生成

### 用途
テキストコンテンツ（投稿、記事、メール等）を生成する

### テンプレート

```
あなたは{platform}の{content_type}を作成する専門家です。

## ターゲット
{target_audience}

## トピック
{topic}

## トーン
{tone}

## 制約
- 文字数: {char_limit}
- 禁止ワード: {prohibited_words}
- 必須要素: {required_elements}

## 出力形式
{output_format}
```

### 使用例

```
あなたはXの投稿を作成する専門家です。

## ターゲット
[YOUR_TARGET_AUDIENCE]

## トピック
[YOUR_NICHE_TOPIC]

## トーン
親しみやすく、共感を誘う

## 制約
- 文字数: 140文字以内
- 禁止ワード: 簡単に稼げる、すぐに
- 必須要素: 具体的な数字、行動喚起

## 出力形式
本文のみ（ハッシュタグなし）
```

---

## パターン2: 分析・評価

### 用途
既存コンテンツを分析・評価する

### テンプレート

```
以下の{content_type}を分析してください。

## 分析対象
{content}

## 分析観点
1. {aspect_1}
2. {aspect_2}
3. {aspect_3}

## 出力形式
### 総評
[全体的な評価]

### 良い点
- [ポイント1]
- [ポイント2]

### 改善点
- [ポイント1]: [改善案]
- [ポイント2]: [改善案]

### スコア
| 項目 | スコア (1-10) |
|------|--------------|
| {aspect_1} | X |
| {aspect_2} | X |
```

---

## パターン3: 改善・リライト

### 用途
既存コンテンツを改善する

### テンプレート

```
以下の{content_type}を改善してください。

## 元のコンテンツ
{original_content}

## 改善の方向性
{improvement_direction}

## 維持すべき点
- {keep_1}
- {keep_2}

## 出力形式
### 改善版
[改善したコンテンツ]

### 変更点
- [変更1]: [理由]
- [変更2]: [理由]
```

---

## パターン4: 構造化・整理

### 用途
情報を整理・構造化する

### テンプレート

```
以下の情報を整理してください。

## 入力情報
{raw_information}

## 整理の観点
- 目的: {purpose}
- 対象: {audience}
- 形式: {format}

## 出力形式
{output_structure}
```

---

## パターン5: ステップバイステップ

### 用途
段階的な思考や実行が必要なタスク

### テンプレート

```
以下のタスクをステップバイステップで実行してください。

## タスク
{task}

## 手順
### Step 1: {step_1_name}
{step_1_instruction}

### Step 2: {step_2_name}
{step_2_instruction}

### Step 3: {step_3_name}
{step_3_instruction}

## 各ステップの出力を明示してください
```

---

## パターン6: ペルソナベース

### 用途
特定のペルソナ視点でのコンテンツ作成

### テンプレート

```
以下のペルソナになりきって{task}を行ってください。

## ペルソナ
- 年齢: {age}
- 職業: {occupation}
- 悩み: {pain_points}
- 願望: {desires}
- 価値観: {values}

## タスク
{specific_task}

## ペルソナの視点で考慮すべきこと
- {consideration_1}
- {consideration_2}
```

---

## パターン7: A/Bテスト生成

### 用途
複数のバリエーションを比較用に生成

### テンプレート

```
以下の条件で{count}パターンの{content_type}を作成してください。

## 基本条件
{base_conditions}

## バリエーションの軸
- パターンA: {variation_a}
- パターンB: {variation_b}
- パターンC: {variation_c}

## 出力形式
各パターンについて:
1. コンテンツ本文
2. 狙い・意図
3. 想定される反応
```

---

## パターン8: フィードバック反映

### 用途
フィードバックを受けて修正する

### テンプレート

```
以下のフィードバックを反映して修正してください。

## 元のコンテンツ
{original_content}

## フィードバック
{feedback}

## 修正方針
- 維持: {keep}
- 変更: {change}
- 追加: {add}
- 削除: {remove}

## 出力形式
### 修正版
[修正したコンテンツ]

### 反映した点
- [フィードバック1] → [対応内容]
```

## 組み合わせのヒント

### 効果的な組み合わせ

| 組み合わせ | 効果 |
|-----------|------|
| 分析 → 改善 | 根拠のある改善 |
| ペルソナ → A/Bテスト | ターゲットに刺さる |
| ステップバイステップ → 構造化 | 複雑なタスクの整理 |
