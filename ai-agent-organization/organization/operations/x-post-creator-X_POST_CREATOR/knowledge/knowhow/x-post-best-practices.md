# X投稿のベストプラクティス

このファイルには、X_POST_CREATORエージェントがX投稿を作成する際のベストプラクティスを記載します。

---

## 基本原則

### 1. 簡潔性を最優先
- 短い文字数で最大の価値を伝える
- 不要な修飾語を削ぎ落とす
- 一文に複数の意味を詰め込まない

### 2. note記事との一貫性
- note記事の核心メッセージを正確に反映
- トーンがnote記事と矛盾しないよう注意
- note記事へのリンクを自然に配置

### 3. ハッシュタグは使用しない
- ユーザー指定の方針により、ハッシュタグは付けない
- 投稿の本質で勝負する

## 投稿タイプ別の戦略

### 値上げセールス投稿（期間限定）
- **目的**: 値上げ前の緊急性を訴求し、即購入を促す
- **構成**: [緊急性の強調] + [商品の価値] + [値上げ情報] + [価値向上ポイント] + [ターゲット明示] + [再度期限強調] + [リンク]
- **文字数**: 250-350字（詳細情報を含むため長め）
- **重要ポイント**:
  - 期限を3箇所以上で強調（冒頭・中盤・最後）
  - 年号表記を避け、「今の価格」「今日まで」など相対的表現を使う
  - 補足コンテンツ追加など、価値向上ポイントを織り込む
  - チェックリスト形式でターゲット層を明示
  - 社会的証明（購入者多数など）は控えめに（画像で補完する場合）
- **例**:
  ```
  【値上げは明日から】

  たった数回の対話で、
  プロンプトなしに100点の成果物を引き出す。

  そんな「AI壁打ち」の全技術を、
  10ヶ条にまとめたnote記事。

  明日（2/1）から3,480円に値上げします。

  今の価格で買えるのは今日が最後。

  しかも、
  最近第2条・第3条の補足動画も追加して、
  さらにボリュームアップしてます。

  正直、動画追加した後に、
  「これで値上げ前価格はやばいな」
  って思いました。

  でも、今日までと決めてるので。

  ✔︎ プロンプト作成に時間をかけたくない人
  ✔︎ AIとの対話で思い通りの成果物を得たい人
  ✔︎ 壁打ちスキルで作業効率を劇的に上げたい人

  こんな人には、間違いなく刺さる内容です。

  続々と手に取っていただいてますが、
  今の価格は今日23:59まで。

  明日からは3,480円になります。

  詳しくはnoteで読めます。
  リンクは下記から。

  [YOUR_NOTE_URL]
  ```

### note宣伝投稿（告知投稿）
- **目的**: note記事への誘導
- **構成**: [フック] + [問題点の指摘] + [解決策への興味喚起] + [リンク]
- **文字数**: 制限なし（内容に応じて柔軟に）
- **重要ポイント**:
  - **🔴 1行目フックが全て**: ターゲットが普段から目にしている言葉・日常の心の声をそのまま1行目に置く
    - 気づき・共感系: 読者の心の声（「AIを勉強しなきゃ」「自分には向いてないのかも」）をそのまま
    - ノウハウ系: 読者が手に入る理想の未来・ベネフィットを具体的に1行目で提示
  - **穴を広げる**: 1行目でグサッと刺したら、同じテーマで「あるある→共感→気づき」を重ねて穴を広げる。最終的に提言→noteで解説という流れ
  - **じらして誘導**: 結論・理由・まとめは書かない。読者が「続きが気になる」と思わせてnoteへ誘導するのがゴール
  - **改行しすぎない**: 一文ごとに改行するのはNG。数行の塊で訴求するイメージ
  - **自然体のトーン**: 「〜なんですよね」「〜だったりします」など押し付けがましくない表現
  - **興味喚起を入れる**: 「じゃあどうやって？」「それがあるんです」のように読者が気になる形に
- **良い例**:
  ```
  【🎁新作note公開のお知らせ】
  [YOUR_NICHE]で成果を出すには、
  「[YOUR_KEY_INSIGHT]」
  という話を書きました！

  「[YOUR_READER_PAIN_POINT]」

  って悩んでる人の9割は、
  [YOUR_ROOT_CAUSE]だけだったりします。

  [YOUR_EXPLANATION]

  じゃあどうやって[YOUR_SOLUTION_HINT]するのか？

  それがあるんです。
  ひと手間するだけで[YOUR_RESULT]できる方法が。

  そのことについても話してます。
  ```
- **悪い例（改行しすぎ）**:
  ```
  「[YOUR_READER_PAIN_POINT]…」

  その原因、[YOUR_WRONG_ASSUMPTION]じゃないかもしれません。

  [YOUR_NICHE]で成果が出ない本当の理由は「[YOUR_ROOT_CAUSE]」。

  [YOUR_ROOT_CAUSE]を整えるだけで、劇的に変わります。
  ```
  → 一文ごとに改行しすぎて文脈が途切れる、心に響かない

### ノウハウ長文投稿（最重要設計原則）
- **最終ゴール**: 「読者が今すぐ動き出したくなること」。情報を正確に届けることとは別物
- **一流の4段構造**（この順序で設計すること）:
  1. **理想の未来を示す** → 読者が「自分もそうなれる」と感じる未来を冒頭で提示
  2. **問題点を指摘する** → 今のままではダメな理由を突きつける
  3. **動かざるを得ない理由を作る** → 「今すぐやらないといけない」という緊迫感
  4. **具体的なフローで手順まで降ろす** → 投稿を読み終えた後すぐ動ける手順
- **感情の崩しも必須**: 有益な情報をフラットに届けるだけでは「へー」で終わる。+αの価値として「クスッと笑える」「自己ツッコミ」「フローで今すぐ動ける」が乗って初めて一流
  - 2〜3段落に一度は「笑」や「この投稿はなんだったんだ笑」のような自己ツッコミを仕掛ける
- **Tipsの羅列はNG**: 「知識を教える」設計ではなく「読者が動ける」設計を最優先に

#### 🔴 冒頭3〜5文「刺し設計」の必須ルール（2026-03-01追加）

冒頭140文字が投稿の成否を決める。1〜2文目だけでなく3文目以降も刺しを維持すること。

**禁止パターン（他人事の観察で始めない）**:
- ❌ 「〇〇する人、めちゃ多いんですよね。」→ 読者を傍観者にさせる
- ❌ 「その差の正体は〜じゃないです。ただ〜なんですよ。」→ 3文目が普通になりすぎている

**意識すること**:
- 1〜2文目がOKでも3文目以降が「普通」になりやすい。冒頭3〜5文を通じて刺しの鋭さを維持する意識を持つ
- 読者が「自分のことだ」と感じているかを書いた後に自問する

### オリジナル投稿
- **目的**: Xプラットフォーム独自の価値提供
- **構成**: [気づき] + [具体例] + [行動喚起（任意）]
- **文字数**: 140-280字（内容に応じて柔軟に）
- **例**:
  ```
  [YOUR_TOOL/SKILL]を使い始めて3ヶ月。
  最初は「便利だな」くらいだったけど、今では業務の50%を[YOUR_TOOL/SKILL]に任せてる。

  一番変わったのは「考える時間」が増えたこと。
  単純作業を[YOUR_TOOL/SKILL]に任せることで、戦略や企画に集中できるようになった。

  [YOUR_TOOL/SKILL]は「考えるため」のツールだと実感してる。
  ```

## トーン調整のポイント

### ユーザー指定のスタイルに対応
- ユーザーが投稿例を与えた場合、そのトーン・リズム・語彙を分析
- 「です・ます調」「だ・である調」「体言止め」など、文体を合わせる
- 絵文字の有無、句読点の使い方も参考にする

### 例: カジュアルスタイル
```
[YOUR_NICHE]楽しすぎる。
[YOUR_PROCESS]まで全部[YOUR_METHOD]と一緒に進めてる。
今までの「一人で悩む時間」が激減した。
```

### 例: フォーマルスタイル
```
[YOUR_NICHE]を活用することで、作業効率が大幅に向上しました。
特に[YOUR_SPECIFIC_USE_CASE]の段階での活用が有効です。
詳細は以下の記事でまとめています。
[リンク]
```

## 文字数の選び方

### 短文（〜140字）
- **適用場面**: note宣伝投稿、インパクト重視の投稿
- **メリット**: 読みやすい、リツイートされやすい
- **注意点**: 情報量が限られる

### 中文（141〜280字）
- **適用場面**: オリジナルコンテンツ、深い気づきを共有
- **メリット**: 情報量が多い、読者の共感を得やすい
- **注意点**: 長すぎると読まれない可能性

### 超長文（500〜750字）
- **適用場面**: note記事からの長文投稿、ノウハウ共有、教育的コンテンツ
- **目標文字数**: 500〜750字（読みやすさと情報量のバランス）
- **メリット**: 深い内容を伝えられる、保存されやすい
- **注意点**: 文脈の流れが重要、長すぎると離脱される
- **重要ポイント**:
  - **リード文を強烈に**: 既視感のない表現で読者を惹きつける（例: 「【〇〇についてシェアします！】」）
  - **文脈をスムーズに**: 前後の文章を意識してつなげる、不自然な接続詞を避ける
  - **「つまり」「でも、」の多用を避ける**: すぐに結論に飛ぶ表現を抑える
  - **自然なつなぎ**: 「その意味で〜」「この仕組みと関連して〜」など
  - **主張の強調は柔軟に**: 3〜5箇所程度、内容に応じて調整OK
  - **🔴 主張（━━━）の直前に「接続詞」、直後に「受け皿」を必ず配置**: 主張の直前には「だからこそ、」「じゃあどうするかっていうと、」等の接続詞で橋をかけ、直後には主張を噛み砕いて読者に着地させる受け皿の文を置く。「接続詞→主張→受け皿」をセットで1構造とし、どちらか片方だけでは不十分
  - **具体例は適度に**: 入れすぎるとくどくなる、必要な箇所に絞る
  - **🔴 リズムのブレイクポイントを2〜3段落に1回仕掛ける（単調になると離脱される）**:
    - **擬音・副詞**: 「ゴリゴリ」「ガンガン」「ズコー」など場面を想起させる表現
    - **具体例・たとえ話**: 「例えば、」で抽象から具体に降りて空気をリセットする
    - **読者の心の声**: セリフ引用で読者を引き込み、そこからひっくり返す
  - **🔴 1文内途中改行で「ためて→落とす」リズムを設計する（2026-03-01追加）**:
    - 長文の段落改行とは別に、1文の途中で意図的に改行を入れる技術を使う
    - 「〜は、」「〜すると、」など主語・状況説明の位置で止め、次行でオチや結論を落とす
    - 読者に息を吸わせる設計。述語・結論を次行に渡すことで「ためる」時間が生まれる
  - **🔴 接続詞1語を独立段落として使う（2026-03-01追加）**:
    - 「そして、」「だから、」などを文頭に置くだけでなく、1語だけの段落として独立させる
    - 前後の文の間に視覚的余白と期待感が生まれ、次の内容を読者が自然に待つ状態になる
    - 多用するとくどくなるため1〜2箇所に限定する
  - **🔴 CTAは1つに絞る（複数詰め込み厳禁）**:
    - note誘導・保存・シェアなど複数のCTAを同時に求めると読者の注意が散漫になる
    - 投稿の目的に応じて「ブクマ」か「note誘導」かを1つだけ選ぶ
    - どちらも入れたい場合でも、優先する1つだけを1行で書いて終わる
  - **全体の流れ**:
    1. リード文で強烈に惹きつける（理想の未来・問題点を示す）
    2. 本当のノウハウ・知識パートに移行
    3. 文脈が途切れないように前後をつなげながら展開
- **例**: [2026-01-31_x-post-ai-prerequisites.md](../../../workspace/2026-01-31_x-post-ai-prerequisites.md) の編集後バージョン参照

- ❌ 過度な煽り文句（誠実性を保つ）
