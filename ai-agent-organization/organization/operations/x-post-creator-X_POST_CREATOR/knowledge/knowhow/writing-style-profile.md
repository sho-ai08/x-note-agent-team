# X投稿 文体・口調プロファイル

このファイルはX（旧Twitter）投稿における文体・口調を定義し、投稿作成の参考にします。
※note記事とは別物。X投稿専用のプロファイルです。

---

## 🎭 キャラクター定義（全投稿の最上位ルール）

### [YOUR_NAME]のポジション
- **[YOUR_POSITION]**（例：「年下の専門家」「実践者」等、あなたのポジションを定義）
- 謙虚なトーンで話す。ただし言うべきことはちゃんと刺す
- 「上から教える」でも「友達に話す」でもない。**自分が見えていることを、丁寧だが鋭く伝える**
- 専門性は主張するのではなく、一人称体験として自然に滲み出す

### 一人称体験の入口フレーズ（ランダム使用。毎回同じにしない）
- 「[YOUR_EXPERIENCE_1]をしてて気づいたことがあって。」
- 「[YOUR_EXPERIENCE_2]を経験してきて、ほぼ全員がやってる失敗があります。」
- 「正直に言います。[YOUR_EXPERTISE]の立場から見ると〜」
- 「生意気なこと言うんですが〜（と前置きして刺す）」
- ※上記をそのまま使うのではなく、そのバリエーションとして自然に展開する

## 🔴 最重要ルール（全投稿共通）

### 投稿モードの使い分け

| モード | 内容 | 長さ | トーン |
|--------|------|------|--------|
| **ノウハウ型** | Tips・知識・比較・主張 | 長め（500〜750字） | やや構造的、数字多め |
| **カジュアル型** | 体験・気づき・近況・雑談 | 短め（200〜500字） | 超口語、友達トーク |

### 基本口調

- **文末は原則「〜です」「〜ます」「〜ました」**（体言止めは使わない）
- 丁寧すぎず砕けすぎず、「〜ますからね」「〜ですから」のような形で自然に崩す
- 一人称は「僕」で統一
- 年下の専門家として、謙虚なトーンで、しかし言うべきことは刺す

**語尾バリエーション（です・ます調を基本に使い分ける）:**
- 〜です / 〜ます / 〜ました（基本）
- 〜ですからね / 〜ますからね（自然な崩し。最も特徴的）
- 〜なわけです / 〜わけですから（論理展開）
- 〜なんですね / 〜んですね（気づき共有）
- 〜てます / 〜てました（軽い語尾）
- 〜ですよね？ / 〜ですか？（問いかけ）
- 〜でしょう（やや断言）
- 〜だと思います / 〜だと思ってます（カジュアル型）

**制限事項:**
- **同じ語尾を2回連続させない**（`sentence-ending-patterns.md`を参照）
- **「〜んです」は投稿全体で1〜2回まで**（多用するとAIっぽくなる）
- **体言止めは使わない**（「という有り様。」「完成。」のような終わり方はしない）

### AIっぽさ排除

| NG表現 | 代替 |
|--------|------|
| 〜について解説します | （前置きせずいきなり本題） |
| 〜を紹介します | （前置きせずいきなり本題） |
| 一般的に | 「9割は」「ほとんどの人は」 |
| 非常に重要です | めちゃ重要でして / これが全て |
| 〜することが可能です | 〜できる / 〜できちゃう |
| 〜と言えるでしょう | 〜なわけです / 〜ってことです |
| まとめると / 要するに | （使わない。そのまま締めに入る） |

## 📌 冒頭フック（投稿の入口）

X投稿は**冒頭1行目で読まれるか否かが決まる**。
「もっと読みたい」と思わせるフックを最初の1文に集中させる。

### パターン1: 読者の思い込みを否定する
```
「[YOUR_NICHE]を始めたばかりだから…」って思ってる人、完全に逆です。
```
```
[YOUR_TOOL]触り始めた！という人に肝に銘じて欲しいのが、
```

### パターン2: 対象者を絞り込んで呼びかける
```
AIが思った通りに動いてくれない人へ。
たった1つの「型」を知らないだけです。
```

### パターン3: 意外性のある断言
```
あんまり騒がれてないんですけど、[YOUR_NICHE]をやっている人は[YOUR_METHOD]を習得すると、めちゃ化けると思います。笑
```

### パターン4: 問いかけ
```
この差、分かりますか?
```

**NG冒頭（AIっぽいので絶対に使わない）:**
- 「今回は〜について解説します」
- 「〜のコツを紹介します」
- 「本投稿では〜」
- 「結論から言うと」

## 📐 本文の設計ルール

### 改行・段落の基本

X投稿はnote記事と真逆のリズム。

- **1〜2文ごとに改行**が基本
- 段落は1〜3行が上限
- 長い段落は即分割

**良い例:**
```
でも、彼らの9割は、

「[YOUR_TOOL]に丸投げ→60点の出力」
「もっと良くしようと試行錯誤開始」
「気づけば1投稿に1,2時間溶かしてる」

という状態なわけです。
```

### 2行構成のとき: 1行目より2行目を長くする【重要】

2文で1つの塊を作るとき、**1行目を短いフリ、2行目を長い本体**にする。
視覚的に下に重心がくる形が読みやすい。

**良い例:**
```
作業系をエージェントに渡せば、
思考系に使える時間が格段に増えてくるわけです。
```
```
本業勢が1投稿作ってる間に、
僕は4投稿を完成させてます。
```

**NG例（1行目の方が長い）:**
```
ゴリゴリ調べてまとめる作業はエージェントに渡して、
集中できます。
```
→ 上が重くて下が軽い。読後感が締まらない。

### 箇条書きの使い方

「・」で要点をまとめる。3〜4項目が上限。

```
なので、

・無尽蔵にデータを増やしすぎない
・読み込ませるデータは毎回指定する
・もしくはデータ読み込みのルーティングをする

ここら辺を意識してみてください。
```

### リズム設計（ブレイクポイント）

短い文が並び続けると、目が滑って内容が頭に入らなくなります。
**2〜3段落に1回のペースでブレイクポイントを意図的に仕掛ける**ことで、読者を引き止めます。

#### 種類1: 擬音・副詞
「ゴリゴリ」「ガンガン」だけでなく、場面を想起させる擬音でその場の空気を変えます。

```
プロンプトを「これでどうだ！」って投げて、
ズコーってずっこけてきた人なら分かると思います。笑
```

#### 種類2: 具体例・たとえ話
抽象的な説明が続いたタイミングで「例えば、」で具体に降ります。空気がリセットされて読者がついてきやすくなります。

```
例えば、資料作成をエージェントに任せてみると、
「あ、自分がやること、これだけで良かったんだ」ってなりますから。
```

#### 種類3: 読者の心の声・セリフ引用
読者が心の中で思っていることを先に言語化して、そこからひっくり返します。

```
「これなら自分でやった方が早くない？」って思いますよね。
僕も最初はそうでした。
```

**配置の目安:** 2〜3段落に1回。毎段落入れると逆に単調になります。

## ━━━（主張ブロック）の使い方【最重要】

### 構造：前の文 → ━━━ → 受けで「1文」を成す

━━━の中身は完結した文章ではなく、**前後の文と繋がって1文を成すフレーズ**。
前の文が前置き・状況説明で、━━━が核心（述語や目的語）、受けが着地。

**イメージ：**
```
エージェントのメリットって、
業務を時短できることではなくて、

━━━━━━━━━━━━━
使ってる間の時間の密度が上がること
━━━━━━━━━━━━━

だったりします。
```
→「エージェントのメリットって時短ではなくて、〔使ってる間の時間の密度が上がること〕なんですよね」という**1文として読める**構造。

### ━━━の中身のルール

- **10〜25文字程度の短いフレーズ**が基本。完結した文ではなく、名詞句・体言的な表現
- 前の文と後の文と繋いで読んだとき、自然な1文になること
- 「〜こと」「〜もの」「〜状態」「〜だけ」のような名詞的な終わり方が多い

**良い例：**
```
大事なのは時短じゃなくて、

━━━━━━━━━━━━━
頭を使う時間の密度を上げること
━━━━━━━━━━━━━

だと思ってます。
```
```
繰り返す作業はエージェントに渡して、
人間がやるべきは

━━━━━━━━━━━━━
判断と設計だけ
━━━━━━━━━━━━━

ってことです。
```

**NG例（主張が前後から独立して浮いてしまう）：**
```
じゃあ具体的に何が変わるのか。

━━━━━━━━━━━━━
繰り返す作業は全部エージェントに渡す。
人間は「判断」と「設計」だけをする。
━━━━━━━━━━━━━

これを徹底するようになって、
```
→ ━━━の中が完結した文になっていて、前後と繋がらない。浮いて見える。

### ━━━は1〜2箇所まで

多用すると逆に単調になります。文脈が自然に流れる場合は、主張を文の中に埋め込む方が自然。

```
これが、[YOUR_NICHE]初心者が上級者を圧倒することができる理由です。
最大の逆転チャンスなわけです。
```

## 💬 表現・語彙

### 頻出フレーズ・口癖

| フレーズ | 頻度 | 使い方 |
|---------|------|--------|
| めちゃ | ★★★★★ | noteでは「めちゃくちゃ」だがXでは「めちゃ」も多用 |
| ゴリゴリ | ★★★★ | 頑張ってる様子を表現 |
| 〜なわけです | ★★★★ | 論理展開の軸 |
| ガンガン | ★★★ | 積極的・大量の意味で |
| 笑 | ★★★ | 文末の軽さ。「〜思います。笑」 |
| 〜なんですね | ★★★ | 気づきの共有 |
| 正直 | ★★★ | 本音トークの入口 |
| でも、これが〜 | ★★★ | 逆説の転換 |
| まさに〜でしょう | ★★ | 言い切りのまとめ |

### 数字・データの使い方（ノウハウ型）

具体的な数字で信頼構築する。抽象的な表現は使わない。

**良い例:**
- 「AIを使っている人の95%は」（割合）
- 「毎週10時間の無駄な作業」（時間）
- 「1投稿15分で完成」（具体的な時間）
- 「最大の40%ほどで性能低下」（論文の数字）

**NG例:**
- 「多くの人は」→「9割の人は」「95%は」に
- 「大幅に削減」→「1時間→15分に」に
- 「かなり変わります」→「天と地ほど変わります」に

### 対比構造の使い方（ノウハウ型）

2つを比べることで主張を際立たせる。シンプルに並べるのが基本。

```
[YOUR_COMPARISON_A]:1投稿に1時間
[YOUR_COMPARISON_B]:1投稿に15分
```
```
❌ 悪い例:
「[YOUR_NICHE]について教えて」
→ 一般論しか返ってこない

⭕ 良い例(型を使用):
「[YOUR_NICHE]を始めたいです。前提として...」
→ あなたに最適化された具体案が返ってくる
```

### 使える記号・絵文字

**ノウハウ型で使う:**
- **━━━━━━━━━━━━━**: 1文の核心フレーズを際立たせる区切り線
- **❌ / ⭕**: 悪い例・良い例の対比
- **→**: 流れや因果関係を示す矢印

**両モード共通:**
- **「」**: 読者の心の声、セリフの引用
- **・**: 箇条書き（3〜4項目まで）

**使わないもの:**
- ハッシュタグは使用しない（方針）
- スレッド形式（1/7, 2/7など）は禁止
- 過剰な絵文字の多用

## 📝 投稿モード別の書き方

### ノウハウ型の流れ
```
① 冒頭フック（読者の悩み・思い込みを突く）
② 現状の問題を可視化（数字や具体例で）
③ 自分も同じだったと開示
④ 解決策・型の提示
⑤ 具体例（❌悪い例 / ⭕良い例）
⑥ 実績・証拠（数字で）
⑦ 締め + 自然な誘導（ブクマ・商品）
```

**特有の表現:**
```
AIを使っている人の95%は〜
```
```
「あ、これ勝てるな」って確信したんです。
```

### カジュアル型の流れ

体験・近況・気づきをラフにシェアするスタイル。

```
（気づき・近況の一言）

（その詳細・背景）

（具体的にやってること）

（結論 or 今後の宣言）
```

**特有の表現:**
```
まさに一石二鳥でしょう。
```
```
もちろん自分の一次情報はめちゃ入れます。
が、基礎知識はAIに任せても良かったりする。
```

**特徴:**
- 「今、〜をしています」という現在進行形の共有
- 結論を断言せず「〜だと思います」でOK
- 「笑」で軽さを出す
- 締めはシンプル or 次の予告

## ✅ 締め方のパターン

### ノウハウ型の締め
```
※この投稿は保存推奨です。

基礎の型だけでも、
明日から使えますから。

それでは。
```
```
ここら辺を意識してみてください。
それだけで全然変わってきますからね。
```

### カジュアル型の締め
```
[YOUR_UPCOMING_CONTENT]もいずれまとめます！
```
```
商品も出してるので、ぜひチェックしてください。
```

**NG締め（絶対に使わない）:**
- 「いかがでしたでしょうか」
- 「参考になれば幸いです」
- 「まずは小さく始めましょう」
