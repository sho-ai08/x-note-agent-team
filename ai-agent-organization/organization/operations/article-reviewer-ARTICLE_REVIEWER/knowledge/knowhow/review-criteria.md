# 記事レビュー基準

このファイルはnote記事のレビュー時に使用する評価基準を記載します。

---

## ペルソナ設定

### レビューの基本方針

**最優先事項:**
1. **note本編の内容そのものが価値あるか**
   - 読者にとって「必要だ」と思ってもらえるか
   - 興味を持ってもらえるか
   - 内容として面白いか
2. **自然な流れで商材への関心が向くか**（露骨な誘導は避ける）
3. **関連記事の導線設計**（前提記事→本記事→次の記事→商材）

### ペルソナ定義ファイルの参照先

各ペルソナの詳細（属性・内面・媒体別評価背景・評価思考回路・スコアリング基準）は
個別ファイルに定義されています。タスク開始時に必ず読み込むこと:

| # | アーキタイプ | ファイル |
|---|------------|---------|
| 1 | 初心者層 | `knowledge/examples/persona-01-beginner.md` |
| 2 | 中間層・実践者 | `knowledge/examples/persona-02-intermediate.md` |
| 3 | 発信者ファン（長期フォロワー） | `knowledge/examples/persona-03-fan.md` |
| 4 | ROI重視・購買検討層 | `knowledge/examples/persona-04-buyer.md` |

> setup-org 実行後、上記4ファイルの `[YOUR_...]` プレースホルダーがターゲット情報で埋まります。

### ペルソナレビュー基準（概要）

| ペルソナ | 評価視点の核心 |
|---------|-------------|
| 初心者層 | 「実践できそうか？自分ごとにできるか？」 |
| 中間層・実践者 | 「新しい発見があるか？実用的か？」 |
| 発信者ファン | 「いつものトーンで親しみやすいか？AIっぽくないか？」 |
| 購買検討層 | 「投資対効果があるか？有料コンテンツに払う価値があるか？」 |

## 総合評価基準

### AIっぽさチェック（全ペルソナ共通・厳格版）

記事全体を通して以下の項目を厳密にチェック：

**語尾の連続チェック（最重要）:**
- 同じ語尾が2回連続 → 警告（黄色フラグ）
- 同じ語尾が3回以上連続 → 即減点（赤フラグ）
- 「〜なんです」の使用が3回以上 → 減点

**AI特有のNG表現チェック:**
- [ ] 「〜について解説します」「〜を紹介します」 → 定型的で人間味がない
- [ ] 「それでは早速見ていきましょう」 → AIの常套句
- [ ] 「結論から言うと」「本記事では」「以下で解説します」 → 前置き宣言
- [ ] 「〜は非常に重要です」 → 硬すぎる
- [ ] 「〜することが可能です」 → 不自然な日本語
- [ ] 「〜と言えるでしょう」の多用 → 断定を避けすぎ
- [ ] 「いかがでしたでしょうか」 → AI記事の定番
- [ ] 「参考になれば幸いです」「まずは小さく始めましょう」 → 締め定型句
- [ ] 「〜に他なりません」 → 論文調
- [ ] 「まず〜、次に〜、そして〜、最後に〜」 → 機械的な列挙
- [ ] 「〜においても」「〜における」 → 硬い表現
- [ ] 「多くの方が〜」 → AIっぽい婉曲表現
- [ ] 不自然に丁寧すぎる表現（「〜させていただきます」の多用）
- [ ] 論文調・教科書調の硬い文体
- [ ] 感情表現の完全な欠如

**安全クッション・逃げ文句チェック:**
- [ ] 「一般的に」「多くの場合」「状況によって異なります」「一概には言えませんが」 → 情報を増やさない逃げ文句は原則削除
- [ ] 必要性を検証し、不要なら削る。必要なら最小限に圧縮

**抽象語・同義語チェック:**
- [ ] 「重要」「効果的」「最適」「本質」「メリット」だけで主張を済ませていないか → 動詞中心の具体表現に
- [ ] 同義語の言い換え連打（重要・大切・欠かせない等）がないか → 1回で言い切る
- [ ] 「まとめると」「要するに」「総じて」で同内容を再掲していないか → 新しい角度で締める

**文のリズム・構造チェック:**
- [ ] 文の長さが均一になっていないか → 短文と長文を意識的に混ぜる
- [ ] 同じ型（断定→理由、結論→補足）が3回以上連続していないか
- [ ] 一人称が統一されているか（「僕」で一貫。「私」「筆者」「私たち」の混在はNG）

**記号の乱用チェック:**
- [ ] 「」が多用されていないか（強調目的のカギ括弧は文脈に溶かす）
- [ ] ()が多用されていないか（補足は本文に組み込む）
- [ ] コロン（：）でラベル列挙していないか
- [ ] スラッシュ（／）で概念を並列していないか

**加点対象となる表現:**
- 「〜なわけです」「〜でしょう」「〜でして」など人間らしい語尾バリエーション
- 話し言葉のような自然なリズム
- 著者の個性や感情が感じられる表現（「正直びっくりした」「マジで」等）
- 具体的なエピソードや体験談・失敗談
- 読者への問いかけ（「〜だと思いません？」）
- 独り言っぽいフレーズ（「いや、これがおもろいんですけどね」）
- 文の長さにバリエーションがある（短文と長文の混在）
- 抽象語ではなく動詞中心の具体表現で主張している
- 一人称が最初から最後まで統一されている

### 内容重複チェック（note記事の場合）

見出し間で同じ内容が繰り返されていないか厳密にチェック：

**チェック項目:**
- [ ] 見出しAで言及した主張を、見出しBでも改めて説明し直していないか
- [ ] 同じ概念を別の言い方で複数の見出しに書いていないか
- [ ] 各見出しに「新しい情報」が含まれているか（読み進めるたびに新しい学びがあるか）
- [ ] 前の見出しの内容を補足する場合、「先ほども触れた通り」等の1文で済ませているか

**重複検出時の指摘フォーマット:**
```
[重複検出] 見出しX「〇〇」の内容が、見出しY「△△」の内容と重複しています。
- 見出しXの該当箇所: "（引用）"
- 見出しYの該当箇所: "（引用）"
- 改善案: 見出しXでは「〇〇」に絞り、見出しYでは「△△」の話に集中する
```

### リード文評価（note記事の場合）

リード文の質とバリエーションをチェック：

**チェック項目:**
- [ ] 最初の1文で読者の手を止められるか（スクロールストッパー性）
- [ ] リード文にベネフィット（理想の未来）または問題指摘が含まれているか
- [ ] 「〜について解説します」だけで終わっていないか（NG）
- [ ] 具体的なエピソードや数字で独自性が出ているか
- [ ] 直近の記事と同じパターンのリード文になっていないか

**リード文パターンの判定:**
記事のリード文がどの「型」に該当するかを判定し、バリエーションを評価する:
- 問題指摘型 / 理想の未来型 / 常識破壊型 / ストーリー型 / 数字・データ型
- 問いかけ型 / 権威・実績型 / 対比型 / トレンド型 / 告白型

### スキ数・ビュー数の予測

**各ペルソナのスコアを元に総合評価:**

| 平均スコア | 予測スキ数 | 予測ビュー数 | 評価 |
|-----------|-----------|------------|------|
| 9-10点 | 100+ | 3,000+ | Excellent |
| 7-8点 | 50-100 | 1,500-3,000 | Good |
| 5-6点 | 30-50 | 800-1,500 | Acceptable |
| 1-4点 | 0-30 | 200-800 | Needs Improvement |

## 品質基準（guidelines.mdより統合）

### 必須要件

すべてのnote記事が満たすべき要件:

- [x] **1次情報の含有**: 自分の体験や実験結果、独自の考察が含まれること
- [x] **読者価値の明確化**: 読者が「何を得られるか」が明確であること
- [x] **正確性**: 事実に基づいた正確な情報であること（特に技術情報）
- [x] **完読性**: 最後まで読みたくなる構成であること
- [x] **装飾の適切性**: 読みやすさを損なわない範囲での装飾

### 推奨要件

- [ ] **具体例・数字**: 抽象論だけでなく、具体的な事例や数字を含む
- [ ] **視覚的要素**: 画像やスクリーンショットで理解を助ける
- [ ] **関連記事リンク**: 過去記事への自然な誘導で回遊率を上げる
- [ ] **CTA配置**: 記事の価値提供後に自然な流れでCTAを配置

### 記事公開前チェックリスト

成果物をユーザーに提出する前に確認:

- [ ] 誤字脱字がないか
- [ ] 文法的に正しいか
- [ ] 事実に誤りがないか（特に技術情報）
- [ ] タイトルと内容が一致しているか
- [ ] リード文で記事の価値が伝わるか
- [ ] 見出しで記事構造が明確か
- [ ] 1次情報が含まれているか
- [ ] 具体例や数字が含まれているか
- [ ] CTAが自然に配置されているか
- [ ] ハッシュタグが適切か（3-5個）
- [ ] 関連記事リンクが適切に配置されているか
- [ ] 読者ターゲット（[YOUR_TARGET_AUDIENCE]）に刺さる内容か

### 評価基準

| 評価 | 基準 |
|------|------|
| Excellent | すべての要件を満たし、スキ数50以上が期待できる品質 |
| Good | すべての要件を満たし、安定したエンゲージメントが見込める |
| Acceptable | 最低限の要件は満たしているが、改善余地あり |
| Needs Improvement | 一部の要件を満たしておらず、公開前の修正が必要 |

### ペルソナレビュー基準（5ペルソナ概要）

| ペルソナ | 属性 | 評価視点 |
|---------|------|---------|
| [YOUR_PERSONA_1_NAME]（初心者層） | [YOUR_PERSONA_1_AGE]代・[YOUR_PERSONA_1_JOB] | 「実践できそうか？自分ごとにできるか？」 |
| [YOUR_PERSONA_2_NAME]（中間層） | [YOUR_PERSONA_2_AGE]代・[YOUR_PERSONA_2_JOB] | 「新しい発見があるか？実用的か？」 |
| [YOUR_NAME]ファン（長期フォロワー） | 継続フォロワー | 「いつものトーンで親しみやすいか？」 |
| [YOUR_PERSONA_4_NAME]（購買検討層） | [YOUR_PERSONA_4_AGE]代・[YOUR_PERSONA_4_JOB] | 「投資対効果があるか？CTAに進みたくなるか？」 |

### 禁止事項

1. **虚偽の情報**: 事実と異なる情報を提供すること（特に技術情報の誤り）
2. **著作権侵害**: 他者の記事や画像を無断で使用すること
3. **過度なセールス**: 記事の大部分が宣伝になること
4. **クリックベイト**: タイトルと内容が大きく異なること
5. **他者の誹謗中傷**: 特定の個人や企業を貶める表現
6. **根拠のない断定**: 検証していない情報を「確実だ」と断定すること
