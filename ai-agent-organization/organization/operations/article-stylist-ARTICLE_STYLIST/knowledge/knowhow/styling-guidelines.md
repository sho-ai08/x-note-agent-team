# note記事装飾ガイドライン

このファイルはnote記事の装飾・CTA挿入・関連記事リンク配置のガイドラインを記載します。

---

## 使用可能なMarkdown

### 許可されているMarkdown
- **##**（H2見出し）
- **>**（引用）
- ****（Bold/太字）

### 禁止されているMarkdown
- H3見出し（###）
- 箇条書き（-, *）
- リンク（[text](url)）※関連記事リンクは例外
- コードブロック（```）
- その他すべてのMarkdown

**重要:** このnote記事では、シンプルさを保つため、使用するMarkdownを最小限に絞っています。

---

## 見出し（##）の使い方

### 基本ルール
- H2見出し（##）のみ使用
- 各見出しは記事の大きな区切りを示す
- 見出しごとに1,000字程度の本文を想定

### 見出し名のスタイル
- 簡潔で分かりやすい
- 読者の興味を引くフレーズ
- 疑問形や断定形を使い分ける

**例:**
```
## そもそも何がしたい研究なの？
## AIが膨大なデータを必要とする理由について
## 「ファインチューニングとしての人生」という捉え方
## 現実と仮想世界は同じじゃない
```

---

## 引用（>）の使い方

### 使用場面
- **重要な主張を強調したい時**
- **記事の核心となるメッセージ**
- **読者に特に覚えてほしいポイント**

### 使用ルール
1. 引用(>)とBold(**太字**)を組み合わせる
2. 段落を独立させる
3. 短い文にする（1-2行）

**良い例:**
```
> **トヨタが研究している＆逆マトリックスのような夢のような研究**
```

```
> **「都市をまるごとGPTで回せる」なら、「SNS投稿に対する反応」だって作り出せるのでは？**
```

**悪い例:**
```
> このようにトヨタが研究している内容は、逆マトリックスのような夢のような研究であり、非常に興味深いものとなっています。
```
→ 長すぎて読みにくい

---

## Bold（太字）の使い方

### 使用場面
- 重要な単語やフレーズを強調
- 固有名詞や専門用語の初出時
- 数字やデータ

### 使用ルール
- **過度な使用は避ける**（1段落に1-2箇所程度）
- 連続して太字にしない
- 文章全体を太字にしない

**良い例:**
```
まず、第1条。

これこそがすべての**土台**です。
```

```
僕たちは **「ゼロから学習」** してるんじゃなくて、**「事前学習済みモデルをファインチューニング」** してるだけ
```

**悪い例:**
```
**まず、第1条。これこそがすべての土台です。**
```
→ 全体が太字で強調の意味がない

## CTAの配置

### 配置場所
- **記事の最後（メイン）**
- 記事の中間（オプション）

### CTAの種類
1. **有料note誘導**（メイン）
2. **メンバーシップ誘導**
3. **関連記事誘導**
4. **フォロー促進**

### CTA作成のルール
1. **見出しを付ける**（## 少しお知らせ）
2. **価値を明確に伝える**（何が読めるのか、何が得られるのか）
3. **行動を促すフレーズで締める**
4. **自然な流れで挿入**（記事の価値提供後）
5. **有料noteのリンクを含める**

**良い例（有料note誘導 - デフォルト）:**
```
## 少しお知らせ

僕のnoteでは、[YOUR_NICHE]についての実践的な知見を定期的に発信しています。

今回のような内容から、さらに踏み込んだ[YOUR_TOPIC]まで、[YOUR_AUTHORITY_BASIS]をベースにお届けしてます。

有料記事やメンバーシップでは、もっと踏み込んだ内容を書いているので、もう一歩深く知りたいと感じた方はぜひ覗いてみてください。

[YOUR_NOTE_URL]

> **いいねを押してもらえると記事執筆のモチベーションに繋がるので、ぜひよろしくお願いします。**
```

**悪い例:**
```
[YOUR_UPSELL_PRODUCT_NAME]やってます。登録してください。
[リンク]
```
→ 価値が伝わらない、唐突

**注意**: CTA内の文章も[YOUR_NAME]の口調（フランクで親しみやすい）を維持すること。記事のテーマに合わせてCTAの導入文を自然に調整する（「今回のような〇〇の考え方から」の部分を記事内容に応じて変える）。

## 関連記事リンクの挿入

### 挿入場所
- 記事の途中で自然な流れで挿入
- 同じテーマの記事への誘導
- 補足情報がある記事への誘導

### 挿入のタイミング
- 「詳しくは別の記事で書いています」
- 「以前、〇〇について書きました」
- 「こちらの記事も参考にしてください」

### フォーマット
```
ちなみに、この話については以前の記事で詳しく解説しています。
[記事タイトル](リンク)
```

## 装飾の全体バランス

### 適切なバランス
- **見出し**: 5-10個程度
- **引用(>)**: 3-5箇所程度
- **Bold**: 各段落に1-2箇所程度

### 避けるべきパターン
- 太字だらけ → 強調の意味がなくなる
- 引用だらけ → うるさくなる
- 見出しが少なすぎる → 読みにくい
- 見出しが多すぎる → 細切れで読みづらい

## 装飾時のトーン保持ルール（重要）

装飾作業で**文章のトーンや表現を壊さない**こと。以下を厳守:

### やってはいけないこと
- 装飾のために文章を書き換えて、AI臭い表現にしてしまう
- 強調のために文を追加して、機械的な言い回し（「〜は非常に重要です」等）を入れる
- 引用ブロックの中身を定型的な表現に変えてしまう
- CTAの文章を硬い丁寧語で書き直してしまう
- 安全クッション（「一般的に」「多くの場合」）を装飾で追加してしまう
- 抽象語（「重要」「効果的」「本質」）だけの強調文を追加してしまう
- 同義語の言い換え連打（重要・大切・欠かせない）で装飾してしまう
- 「」や()を装飾目的で乱用してしまう

### 意識すべきこと
- 装飾はあくまで「既存の文章を際立たせる」作業であり、文章自体を変更する作業ではない
- CTAの文言も、[YOUR_NAME]の口調（フランクで親しみやすい）を維持する
- 装飾後に読み直して「AIが書いた感じ」になっていないか確認する
- 語尾が連続していないか、装飾で追加した文が原因で語尾パターンが崩れていないかチェック
- 一人称が「僕」で統一されているか確認する（装飾で「私」等を混入させない）
- 最終仕上げ時は `shared/knowledge/anti-ai-rewrite-master.md` も参照する

## note記事の構成テンプレート（guidelines.mdより統合）

### 基本構成

```
1. タイトル（25-35文字推奨）
   - 検索されやすいキーワードを含む
   - 読者の課題を明確にする

2. リード文（3-5行）
   - この記事で何が得られるかを明示
   - 読者の興味を引くフックを入れる

   【重要】リード文の3要素:
   ① ベネフィット（理想の未来）
   ② 問題点の指摘（現状の課題）
   ③ 信頼性（なぜ私が語れるのか）

3. 本文
   - H2見出しで大きく区切る
   - 3-4行ごとに改行を入れ、読みやすくする
   - 太字・引用を適度に使う（多用厳禁）
   - 1つの見出し（H2）配下に最低300-500文字は書く

4. まとめ
   - 記事の要点を3-5点で整理
   - 読者へのアクションを提案

5. CTA（Call To Action）
   - 有料note誘導 / 関連記事リンク / フォロー促進

6. ハッシュタグ（3-5個）
```

## 文章スタイル基準（guidelines.mdより統合）

### コミュニケーションスタイル
- [YOUR_NICHE]の専門知識を分かりやすく伝える
- 堅苦しくなりすぎず、読者が親しみを感じる語り口
- 「です・ます」調で統一
- 読者に語りかけるスタイル

### 推奨する表現
- **能動態を使う**: 「実験しました」（○）vs「実験された」（×）
- **具体的な数字や例を挙げる**: 「効率が上がる」（×）vs「作業時間が30%短縮」（○）
- **短い文を心がける**: 1文は60文字以内が目安
- **結論を先に述べる**: PREP法を活用

### 避ける表現
- 曖昧な表現（「〜かもしれない」「おそらく」）
- 冗長な表現（「〜することができる」→「〜できる」）
- 安全クッション（「一般的に」「一概には言えませんが」）
- 抽象語の空回り（「重要」「効果的」「最適」だけで押し切らない）
- 同義語の言い換え連打
- 前置き宣言（「結論から言うと」「本記事では」）
- 締め定型句（「いかがでしたでしょうか」「参考になれば幸いです」）
- 記号の乱用（「」・()・：・／ を多用しない）

### 読点・接続詞・語尾
- 読点は1文に2つまでを目安
- 接続詞は論理関係が明確に必要な場合のみ
- 同じ語尾は連続2回まで。3回目は必ず変える
- 見出しに絵文字や記号を使わない
