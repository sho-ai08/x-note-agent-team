## 無料記事末尾の有料誘導文テンプレート

無料記事には必ず以下のいずれかのパターンで有料誘導文を挿入する。

### パターンA：問題解決提示型

```
ここまで読んで「じゃあ、具体的にどうすればいいの?」と思った方へ。

[有料記事タイトル]では、今回の内容をさらに深掘りし、
あなたが実際に手を動かせるレベルまで落とし込んでいます。

[YOUR_VALUE_PROPOSITION]を求める全員に読んでほしいnoteです。

▶ [有料記事タイトル]を読む
[YOUR_NOTE_URL]
```

**入口文言のバリエーション（記事テーマに応じて選択）：**
- 問題→解決型: 「じゃあ、具体的にどうすればいいの?」
- 目的→手段型: 「目的は決まった。じゃあ次は何を学べばいいの?」
- 基礎習得型: 「目的は決まって基礎的な所は絶対に抑えたい!」

### パターンB：仕組み→実践の橋渡し型（簡潔版）

```
ここまでが仕組みの理解です。実際の案件でどう使うかのテンプレートと実例を有料記事に置きました。

▶ [有料記事タイトル]を読む
[YOUR_NOTE_URL]
```

### パターンC：状態変化訴求型

```
「使える」から「成果を出せる」に変わりたい方へ。

[有料記事タイトル]は、そのための地図です。
独り立ちして、AIで安定した成果を出し続けられるようになるまで、伴走します。

▶ [有料記事タイトル]を読む
[YOUR_NOTE_URL]
```

**デフォルトリンク**: [YOUR_NOTE_URL]

## CTA配置の実装基準

### 無料記事の場合

| 位置 | 配置内容 | 理由 |
|------|---------|------|
| 記事中盤（価値提供後） | 有料記事への軽い言及 | 読者が「もっと知りたい」と思ったタイミングで動線を提供 |
| **記事末尾** | 有料誘導文テンプレート（A/B/Cから選択） | 必須。無料記事は有料記事のセールスレターとして機能する |
| 末尾・有料誘導文の後 | 回遊設計の3点リンク | 有料に行かない読者に次の無料記事を提供 |

### 有料記事の場合

| 位置 | 配置内容 |
|------|---------|
| 記事末尾 | [YOUR_UPSELL_PRODUCT]への誘導 |

**[YOUR_UPSELL_PRODUCT]誘導テンプレート**:
```
# さらに深く学びたい方へ: [YOUR_UPSELL_PRODUCT_NAME]

この記事で解説した内容を、実際に使いこなせるようになるには、
「継続的な学び」と「疑問を解消できる環境」が必要です。

[YOUR_UPSELL_PRODUCT_NAME]では、
- [YOUR_UPSELL_VALUE_1]
- [YOUR_UPSELL_VALUE_2]
- [YOUR_UPSELL_VALUE_3]

といった環境を提供しています。

▶ [YOUR_UPSELL_PRODUCT_NAME]の詳細・入会はこちら
[YOUR_UPSELL_URL]
```

## 回遊設計の実装基準

### 3点リンクの配置ルール

記事の末尾には必ず以下の3パターンのリンクを配置する。

```
---

**この内容の前提になる話はこちら**
→ [前の記事タイトル](記事URL)

**この内容の続き・実践編はこちら**
→ [次の記事タイトル](記事URL)

**このシリーズの全記事はこちら**
→ [シリーズ一覧タイトル](記事URL)
```

### 配置タイミング

- 有料誘導文の**後**に回遊リンクを置く（有料誘導文が先）
- 有料記事の場合: [YOUR_UPSELL_PRODUCT]誘導の**後**に回遊リンクを置く

### シリーズ記事の場合の例

```
---

**第1回はこちら**（まだ読んでいない方へ）
→ AIの出力がブレる理由：ほとんどの人が知らない原因

**次の回はこちら**
→ 修正指示を何度出しても直らない問題の解決策

**このシリーズの全5回はこちら**
→ AIツールを仕事で使いこなすための5ステップ｜シリーズ一覧
```

## CTA・回遊設計の注意事項

1. **強引な誘導は禁止**: 「今すぐ買ってください」「絶対に読むべき」などの強制的な表現は使わない。「置きました」「まとめました」程度のトーンで
2. **有料誘導文のA/B/C選択基準**:
   - 記事が「問題提起型」→ パターンA（問題解決提示型）
   - 記事が「解説型」→ パターンB（仕組み→実践の橋渡し型）
   - 記事が「体験談型」→ パターンC（状態変化訴求型）
3. **回遊リンクが未整備の場合**: リンク先が存在しない場合はプレースホルダーを置く（記事公開後に追記する想定）
4. **有料記事にも無料誘導文は不要**: 有料記事の末尾は[YOUR_UPSELL_PRODUCT]誘導のみ
