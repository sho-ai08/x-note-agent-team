# noteデータ分析手法

このファイルはnoteアカウントのパフォーマンスデータを分析する手法を記載します。

---

## データ収集方法

### Playwright（Chromeブラウザ操作）を使用

noteのダッシュボードにアクセスして、以下のデータを収集します。

**収集するデータ:**
1. **記事ごとのスキ数**
2. **記事ごとのビュー数**
3. **記事ごとのコメント数**
4. **公開日時**
5. **記事タイトル**
6. **記事URL**

---

## 分析項目

### 1. 記事パフォーマンス分析
- どの記事が最もスキを獲得しているか
- どの記事が最もビューを獲得しているか
- スキ数/ビュー数の比率（エンゲージメント率）

### 2. テーマ別分析
- どのテーマの記事が人気か
  - プロンプト系
  - AI活用系
  - 技術解説系
  - 論文解説系
  - 思考・哲学系
- ターゲット読者層ごとの反応の違い

### 3. タイトル分析
- どんなタイトルが伸びているか
- 文字数（25-35文字推奨）
- フック（引き）の有無
- 数字の使用

### 4. 公開時間・曜日分析
- どの時間帯に公開した記事が伸びているか
- どの曜日が反応が良いか

### 5. トレンド分析
- 時系列での変化
- 最近伸びている記事の傾向
- 過去記事との比較

# X Analytics データ分析手法

このセクションはユーザーが提供するX(Twitter)のanalytics CSVデータを分析する手法を記載します。

---

## データ提供方法

ユーザーが手動でCSVをエクスポートして提供する。
- 標準的な保存先: `ai-agent-organization/inbox/account_analytics_content_YYYY-MM-DD_YYYY-MM-DD.csv`
- CSVはX(Twitter)のAnalytics画面からダウンロードしたもの

## CSVカラム定義

| カラム名 | 内容 | 分析用途 |
|---------|------|---------|
| ポストID | 投稿の一意ID | 識別 |
| 日付 | 投稿日 | 曜日分析・時系列 |
| ポスト本文 | 投稿テキスト | テーマ分類・投稿タイプ判定 |
| インプレッション数 | 表示回数 | リーチ指標 |
| いいね | ハートの数 | 共感度指標 |
| エンゲージメント | 総エンゲージメント数 | 総合的な反応量 |
| ブックマーク | 保存数 | 情報価値指標 |
| 新しいフォロー | 投稿経由の新フォロワー | フォロワー獲得力 |
| 返信 | リプライ数 | 対話促進度 |
| リポスト | RT数 | 拡散力 |
| プロフィールへのアクセス数 | プロフィールクリック | 興味・関心度 |
| URLのクリック数 | リンククリック数 | note誘導力 |

## 投稿タイプ分類ロジック

優先度順に判定:
1. **リプライ投稿**: ポスト本文が`@`で始まる
2. **URL共有のみ**: ポスト本文がURLのみ（他テキストなし）
3. **note告知投稿**: 【...】の告知文がある or note/t.coリンクと告知テキストが共存
4. **通常コンテンツ投稿**: 上記以外

※ 分析対象は主に「通常コンテンツ投稿」と「note告知投稿」。リプライ・URL共有は参考値。

## 主要分析指標

### 1. エンゲージメント率（最重要指標）
```
エンゲージメント率 = エンゲージメント / インプレッション × 100
```
- X全体の平均: 0.5〜1.0%程度
- 2%以上: 優秀
- 5%以上: 非常に優秀

### 2. note誘導率
```
note誘導率 = URLクリック数 / インプレッション × 100
```

### 3. フォロワー獲得率
```
フォロワー獲得率 = 新しいフォロー / インプレッション × 100
```
