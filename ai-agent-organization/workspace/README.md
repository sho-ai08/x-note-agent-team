# Workspace - ログとシンプルな保管場所

エージェントが作成した**成果物の保管場所**です。
作業ログ、生成コンテンツ、一時ファイルなど、すべての成果物をフラットに保存します。

---

## 🎯 Workspaceの位置づけ

- **ログ**: エージェントの作業記録・成果物の履歴
- **シンプル**: フラットな構造で検索しやすく
- **保管場所**: 複雑な階層構造は作らず、ファイル名で判別

---

## 📝 使い方

1. **すべての成果物はここに保存**
   - X投稿、note記事、プロンプト、計画書など
   - エージェントが作成したコンテンツは自動的にここへ

2. **ファイル名で内容を判別**
   - 日付プレフィックス推奨: `YYYY-MM-DD_description.md`
   - 分かりやすい命名を心がける

3. **不要になったら削除OK**
   - 定期的にクリーンアップ
   - 重要なものは別途バックアップ

---

## 📂 ファイル命名例

```
2026-01-10_x-post-ces-ai-trend.md        # X投稿(CESのAIトレンド)
2026-01-10_note-article-draft.md         # note記事の下書き
2026-01-10_strategy-planning-notes.md    # 戦略計画メモ
2026-01-10_prompt-improvement-log.md     # プロンプト改善記録
planning-session-notes.md                # 一時的な計画メモ
```

**ルールは緩く、わかりやすければOK**

---

## 🔄 自動保存について

主要なエージェント(NOTE_CREATOR, X_POST_CREATOR, ARTICLE_STYLIST等)は、成果物を自動的にこのworkspaceに保存します。
手動で場所を指定する必要はありません。
