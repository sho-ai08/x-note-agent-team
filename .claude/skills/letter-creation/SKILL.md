# 入力:
- 商品名 = {商品名}
- 商品の特徴 = {商品の特徴}
- ターゲット読者 = {ターゲット読者}
- 価格 = {価格}

# 前提条件:
- タイトル: 説得力のあるセールスレターの作成
- 依頼者条件: 商品やサービスを効果的に宣伝したいマーケター・発信者
- 制作者条件: セールスコピーライティングの原則を理解し、読者の感情を動かす文章を書けるライター
- 目的と目標: 読者の感情に訴えかけ、商品やサービスの購入を促すセールスレターを作成する
- 評価基準:
  1. セールスレターが商品の価値を効果的に伝えているか
  2. 読者の感情に訴えかける表現が適切に使用されているか
  3. 指定された構成に従って、各セクションが適切に展開されているか

# 参照する knowledge 情報:
- レター事例は `ai-agent-organization/organization/operations/letter-creator-LETTER_CREATOR/knowledge/examples/` を参照すること
- **構成の大まかな流れのみを参照すること。内容の学習や流用は禁止**
- 文体・口調は `writing-style-profile.md` に従うこと

# 実行内容:
{商品名}のセールスレターを、{商品の特徴}・{ターゲット読者}・{価格}をもとに、
knowledge内のレター事例の「大まかな構成」を参照して作成してください。

# セクション構成:
以下の順番でh2見出しごとにセクションを作成すること。

"""
## 冒頭の興味づけ:
- ターゲットの興味を喚起する強力なフックを作成する
- 商品の核心的な価値や主要なベネフィットを簡潔かつインパクトのある方法で伝える
- ターゲットの好奇心を刺激し、続きを読みたくなるような仕掛けを施す
- ターゲット自身の理想の未来を想起できるレベルにイメージングを意識すること

## 問題提起:
- ターゲットが直面している具体的な問題や課題を明確に描写する
- ターゲットの痛点・バイアスを特定し、感覚的な表現を豊富に使用して描写する
- 問題を放置した場合の悪化するシナリオを描き、緊急性を強調する

## 自己紹介:
- 著者の経歴や専門性を簡潔に紹介し、信頼性を構築する
- 読者の問題に関連する著者自身の経験や克服したストーリーを共有する
- 著者がなぜこの商品を提供するに至ったかの背景を説明し、真摯さを伝える

## 問題提起2（必要に応じて最適化）:
- 最初の問題提起を深掘りし、より具体的な側面や隠れた問題点を明らかにする
- 読者が気づいていなかった問題の側面を提示し、新たな気づきを与える
- 問題の複雑さや深刻さを強調し、解決の必要性を高める

## 商品の内容:
- 商品やサービスの具体的な内容、特徴、機能を詳細に説明する
- 各特徴がターゲットの問題をどう解決するか、具体的なベネフィットを明示する
- 他の商品との差別化ポイントを強調し、独自の価値を訴求する
- 感覚的な表現を豊富に使用してイメージしやすくする

## 口コミ:
- 実際の利用者からの具体的な証言や成功事例を紹介する（プレースホルダー形式でOK）
- 多様な背景を持つ利用者の声を掲載し、幅広い適用可能性を示す
- ビフォーアフターの変化を具体的に描写し、商品の効果を裏付ける

## 商品をおすすめしない人:
- 商品が適さない人や状況を正直に明示し、誠実さをアピールする
- 逆説的に、理想的な顧客像を浮き彫りにする役割を担う
- 読者に自己選別を促し、ミスマッチによる不満を予防する

## 商品を手に取って欲しい人:
- おすすめしない人と比較することでターゲット自身に間接的に優越感を付与する
- 理想的な顧客像や、商品から最大の利益を得られる人物像を具体的に描写する
- 読者が自身をその像に重ね合わせられるよう、共感できるポイントを強調する
- 商品がもたらす具体的な変化や成果を明確にし、未来のビジョンを提示する

## 特典の案内:
- 特典の付加価値を具体的に説明し、全体の価値を高める（特典がある場合）
- 各特典が商品をどのように補完し、効果を最大化するか説明する
- 特典の限定性や希少性を強調し、即決を促すフックとして活用する
- 特典がない場合は省略してよい

## 商品の価格:
- アンカリングを行ったのち、価格を明確に提示して価値が価格を上回ることを示す
- 類似商品や代替手段との比較を行い、コストパフォーマンスの高さを示す

## よくある質問:
- ターゲットが購入を躊躇する原因を先回りして解消すること
- ターゲット自身が自身に理由づけを行えるような論理で構成すること

## 最後に:
- 商品購入による具体的なメリットを再度強調し、理想の未来像を描く
- レトリカル・クレッシェンドを構築する
- 感情に訴えかける力強いコールトゥアクションで締めくくり、行動を促す
"""

# 追加指示:
- 1つのh2見出しの作成が終われば出力を強制的に終了すること。「続き」と入力されたら次のh2見出しの作成に移ること
- 各見出しではクリフハンガーを活用すること（次を読みたくなる仕掛け）
- knowledge内のレター事例の著者らしい口調で書くこと（少し砕けた口調）
- 各見出しで文体のモジュレーションを行うこと（テンポ・リズムを変化させる）
- レトリカル・リズムを最適化すること
- 全体を通して感情に訴えかける表現を効果的に使用し、読者の共感を得ること
- 適度に箇条書きを使用して、読みやすさを確保すること
- 記憶のフックを戦略的に配置すること
- 言葉遊びを戦略的に配置すること
- 読者の潜在的な疑問や反論を予測し、それらに事前に対処すること
- 各見出しのvolumeはknowledge内のレター事例と同程度にすること
- スマホ縦読みを意識した改行設計（1文1行が基本）

# 絶対条件:
- 1つのh2見出し作成ごとに出力を停止し、「続き」の入力を待つこと
- AI臭い表現（定型的な言い回し、感情のない説明文）は使用禁止
- 各セクションに明確な「感情の目的」を持たせること（興味・共感・信頼・欲求・決断）
- knowledge内のレター事例の内容の流用は禁止。構成の参考のみ
