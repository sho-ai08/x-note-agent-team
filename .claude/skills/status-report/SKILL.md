# 組織ステータスレポートスキル

ユーザーが組織の状態確認を依頼しました。以下のフローで実行してください。

---

## Phase 1: 組織情報の収集

### 1-1. 組織設定の読み込み

`ai-agent-organization/system/config.yaml` を読み込み、プロジェクト名と目的を取得。

### 1-2. 全エージェントのSTATUS.yaml収集

Globで以下を検索：
```
ai-agent-organization/organization/**/STATUS.yaml
```

各STATUS.yamlから以下を抽出：
- `codename` - エージェント名
- `role` - 役割
- `statistics.total_tasks` - 総タスク数
- `statistics.approval_rate` - 承認率
- `statistics.current_streak` - 連続承認数
- `skills` - スキル一覧

---

## Phase 2: レポート生成

以下のフォーマットでレポートを生成：

```
# 組織ステータスレポート
📅 [今日の日付]

## 組織概要
- **プロジェクト**: [プロジェクト名]
- **目的**: [目的の要約]
- **総エージェント数**: [数]

---

## エージェント一覧

### [アイコン] [エージェント名] - [役割]
| 項目 | 値 |
|------|-----|
| タスク | [total_tasks]件 (承認率: [approval_rate]%) |
| ストリーク | [current_streak]連続 |

**スキル**: [スキル名], [スキル名], ...

---
[次のエージェント...]
```

### アイコン割り当て

| エージェント | アイコン |
|-------------|---------|
| BOSS | 📋 |
| GENESIS | 🎨 |
| ARCHITECT | 🏗️ |
| その他 | 🤖 |

---

## Phase 3: サマリー統計

全エージェントを集計したサマリーを追加：

```
## 組織サマリー

| 指標 | 値 |
|------|-----|
| 総タスク完了数 | [全エージェントのtotal_tasks合計] |
| 平均承認率 | [全体の平均]% |
| 最多タスクエージェント | [名前] ([数]件) |
| 最長ストリーク | [名前] ([数]連続) |
```

---

## Phase 4: 推奨アクション

組織の状態に基づいて推奨アクションを提案：

### 判定ルール

1. **承認率が80%未満のエージェントがいる場合**
   → 「[エージェント名]の承認率が低めです。フィードバックを確認し改善を検討」

2. **タスク数が0のエージェントがいる場合**
   → 「[エージェント名]がまだ稼働していません。タスクを割り当てましょう」

3. **ストリークが10以上のエージェントがいる場合**
   → 「[エージェント名]が好調です。より難易度の高いタスクを任せてみましょう」

```
## 推奨アクション

- [推奨1]
- [推奨2]
- ...
```

---

## Phase 5: 完了

レポートを出力して完了。追加の詳細が必要か確認：

「詳しく見たいエージェントはありますか？名前を教えてください」
