# コンテンツ案出しプロンプト

## 概要
発信者のプロフィール情報からターゲットペルソナを設計し、コンテンツ案を提案するプロンプト。
NOTE_CREATORが新しい記事テーマを検討する際に使用。

---

# 入力:
- 発信者情報 = {発信者情報}

# Knowledge参照:
- `ai-agent-organization/shared/knowledge/noteの心得.md`: note公式方針
- `ai-agent-organization/organization/operations/note-creator-NOTE_CREATOR/knowledge/knowhow/writing-style-profile.md`: 著者の文体プロファイル
- `ai-agent-organization/organization/operations/note-creator-NOTE_CREATOR/knowledge/knowhow/strategy-brief.md`: ターゲット・ポジショニング

# 前提条件:
- タスク: 発信者のプロフィール情報からターゲットペルソナを設計し、コンテンツ案を提案する
- 依頼者条件: 発信者のプロフィールからターゲットペルソナを設計し、コンテンツ案を考えたいマーケター
- 制作者条件: マーケティングやユーザーエクスペリエンスの基本知識を持ち、ペルソナ分析、コンテンツ作成の経験があること
- 目的と目標: 発信者のプロフィールデータを解析して、明確で実用的なターゲットペルソナを設計し、最適なコンテンツ案を提案する

# 変数設定:
- 発信者情報 = "
・発信ジャンル情報 = {発信者のジャンル情報}
・発信のターゲット層 = {発信のターゲット層}
・発信者の実績情報 = {発信者の実績情報}
・過去に出した商品(任意) = {過去に出した商品}
"

# 実行内容:
{発信者情報}から詳細なターゲットペルソナの設計を行い、
ターゲットペルソナの悩み(インサイト)を10個抽出して、
悩み(インサイト)を解決するコンテンツ案を提案してください。

なお、{ステップ形式}に従って、ステップバイステップで実行してください。

# step形式:
それぞれのstepが終われば、「次に進んでも良いですか？」と強制的に質問してください。

## step1 ペルソナ設計:
{発信者情報}からターゲットユーザー像を明確にするためのペルソナ設計を行ってください。
性別、年齢、職業、価値観、行動特性、購買動機などの基本属性に加え、心理的インサイトやライフスタイルも含めて、構造的に整理して詳細に設計してください。

### step1の追加指示:
- `ai-agent-organization/organization/operations/note-creator-NOTE_CREATOR/knowledge/knowhow/strategy-brief.md` を参照し、組織のターゲット層と整合させる
- コードブロック形式で出力

## step2 悩み(インサイト)抽出:
[step1]で設計したペルソナが抱えている悩み(インサイト)を詳細に10個挙げてください。
各悩み(インサイト)は強制的に3文で構成して超具体的にインサイトを少し砕けた口調で描写してください。

### step2の出力フォーマット:
```
- [悩み①の具体的な描写（3文で構成）]
- [悩み②の具体的な描写（3文で構成）]
...（10個まで列挙）
```

## step3 コンテンツ案提案:
[step2]で出力したペルソナが抱えている悩みを解決するためのコンテンツ案を、詳細かつ精緻な案を3つ提案してください。

### step3の追加指示:
- 少し砕けた口調で書いてください
- 内容や希少性に応じて適切な値段設定を行なってください（無料記事の場合は「無料」と記載）
- コンテンツは多様な種類を挙げてください
- コンテンツの内容の各構成要素は、コンテンツに盛り込む内容を構造的にそれぞれ詳細に書いてください
- 提案後、「どのコンテンツ案で進めますか？」と質問すること

### step3の出力フォーマット:
```
◆ペルソナの悩みを解決するコンテンツ案

- コンテンツ案①
記事タイトル：「[タイトル]」([価格])
内容：[内容の詳細説明]
構成：
①[章編]
②[章編]
③[章編]

- コンテンツ案②
...
```

## step4 次のアクション:
選択されたコンテンツ案をもとに、次のアクションを提案してください。

```
【次のアクション】
1. NOTE_CREATORが直接コンセプト設計・見出し構成を作成
2. `note-creation` スキルで本文を執筆
3. ARTICLE_REVIEWERにレビュー依頼
```

# 絶対条件:
- 指示の復唱や余計な補足は入れないでください
- 組織の `shared/knowledge/` を参照して整合性を確保すること
