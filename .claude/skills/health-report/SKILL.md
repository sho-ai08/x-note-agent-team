# 健全性診断レポートスキル

ご主人様よりプロジェクトの健康診断のご依頼を承りました。6つの指標で総合的に診断いたします。

---

## 入力変数

| 変数名 | 説明 | 必須 | デフォルト値 |
|--------|------|------|-------------|
| `{target_path}` | 診断対象ディレクトリ | No | プロジェクトルート |
| `{previous_report}` | 前回レポート（比較用） | No | なし |
| `{detail_level}` | 詳細度（summary / standard / detailed） | No | standard |

---

## Phase 1: データ収集

### 1-1. 知識ベースの確認

- `ai-agent-organization/organization/OVERSEER/knowledge/knowhow/strategy-brief.md` ← 監査基準
- `ai-agent-organization/organization/OVERSEER/learnings.md` ← 過去の学び

### 1-2. プロジェクト全体のスキャン

Glob/Bashを使い、以下のデータを収集:

- 全ファイル一覧と更新日時
- ディレクトリ構造
- エージェント一覧（`.claude/agents/*.md`）
- スキル一覧（`.claude/skills/*/SKILL.md`）
- knowledge/ファイル一覧
- workspace/成果物一覧

---

## Phase 2: 6指標スコアリング

### 指標1: ファイル完全性（20点）

各エージェント・スキルに必要なファイルが揃っているかを評価。

| チェック項目 | 配点 |
|-------------|------|
| 全エージェントに .claude/agents/[NAME].md がある | 5 |
| 全エージェントに learnings.md がある | 5 |
| 全エージェントに HISTORY.md がある | 5 |
| 全スキルに SKILL.md がある | 5 |

### 指標2: 構造一貫性（20点）

ディレクトリ構造が規約に準拠しているかを評価。

| チェック項目 | 配点 |
|-------------|------|
| organization/ 配下が規約通り | 5 |
| .claude/agents/ の命名規則準拠 | 5 |
| .claude/skills/ の命名規則準拠 | 5 |
| shared/ の構造が適切 | 5 |

### 指標3: 清潔度（20点）

不要ファイル・空ディレクトリがないかを評価。

| チェック項目 | 配点 |
|-------------|------|
| .DS_Store等の不要ファイルがない | 7 |
| 空ディレクトリがない | 7 |
| *.bak, *.tmp等がない | 6 |

### 指標4: 鮮度（15点）

ファイルの更新頻度を評価。

| チェック項目 | 配点 |
|-------------|------|
| learnings.mdが30日以内に更新されている | 5 |
| HISTORY.mdが30日以内に更新されている | 5 |
| workspace/に直近30日の成果物がある | 5 |

### 指標5: 重複排除度（15点）

重複ファイル・重複内容がないかを評価。

| チェック項目 | 配点 |
|-------------|------|
| 同名ファイルの重複がない | 5 |
| 同一目的のファイルが分散していない | 5 |
| 設定の二重管理がない | 5 |

### 指標6: ドキュメント充実度（10点）

ドキュメントの整備状況を評価。

| チェック項目 | 配点 |
|-------------|------|
| CLAUDE.mdが最新状態 | 4 |
| knowledge/knowhow/strategy-brief.mdが各エージェントにある | 3 |
| shared/knowledge/が活用されている | 3 |

---

## Phase 3: 総合判定

### スコア算出

6指標の合計点（100点満点）で総合判定:

| ランク | スコア | 説明 |
|--------|--------|------|
| S | 90-100 | 極めて健全。模範的な状態 |
| A | 75-89 | 良好。軽微な改善余地あり |
| B | 60-74 | 普通。いくつかの改善が望ましい |
| C | 40-59 | 要改善。放置すると品質低下のリスク |
| D | 0-39 | 要緊急対応。構造的な問題あり |

---

## Phase 4: 改善ロードマップ

スコアに基づき、改善ロードマップを3段階で生成:

```
## 改善ロードマップ

### 短期（今すぐ対応）
- [緊急度の高い改善項目]
- 所要時間目安: [見積もり]

### 中期（1-2週間以内）
- [品質向上に寄与する改善項目]
- 所要時間目安: [見積もり]

### 長期（1ヶ月以内）
- [構造的な改善項目]
- 所要時間目安: [見積もり]
```
