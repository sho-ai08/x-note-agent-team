# タスク依頼スキル

ユーザーがタスクの実行を依頼しました。BOSSエージェントとして以下のフローで実行してください。

---

## Phase 1: タスク内容の確認

ユーザーが既に詳細を伝えている場合は直接Phase 2へ。
そうでなければAskUserQuestionで確認：

- **タスク内容**: 何をしてほしいか
- **期待する成果物**: どんなアウトプットが欲しいか
- **優先度**: 高・中・低（デフォルトは中）

---

## Phase 2: ルーティング分析

### 2-1. キーワードマッチング

タスク内容を分析し、以下のルーティングルールで最適なエージェントを特定：

| キーワード | エージェント | 理由 |
|-----------|-------------|------|
| 組織、戦略、方針、全体最適、リソース配分、計画、連携、チーム、調整、進捗 | BOSS | チーム統括・戦略判断が必要 |
| エージェント作成、新しいエージェント、人格設計 | GENESIS | エージェント設計が必要 |
| プロンプト、改善、テスト、変数、出力形式 | ARCHITECT | プロンプト設計が必要 |

### 2-2. タスクタイプ判定

| タスクタイプ | 説明 | デフォルト担当 |
|-------------|------|--------------|
| strategic | 戦略・方針に関わる | BOSS |
| planning | 計画・調整に関わる | BOSS |
| creation | 新規作成に関わる | GENESIS |
| improvement | 改善・最適化に関わる | ARCHITECT |
| execution | 実際の作業タスク | 実働部隊 |

### 2-3. 実働部隊への振り分け

`execution` タイプのタスクは、`ai-agent-organization/organization/operations/` 配下のエージェントを検索。

各エージェントの `.claude/agents/` 配下のエージェントMDから専門領域とスキルを読み取り、タスク内容との適合度が最も高いエージェントを選択。

該当エージェントがいない場合は、GENESISに新規エージェント作成を提案。

---

## Phase 3: タスク分解（必要な場合）

タスクが複雑な場合（複数ステップ、複数エージェント必要）は分解：

```
## タスク分解

### 元タスク
[ユーザーの依頼内容]

### サブタスク
1. [サブタスク1] → 担当: [エージェント名]
2. [サブタスク2] → 担当: [エージェント名]
3. [サブタスク3] → 担当: [エージェント名]

### 実行順序
[依存関係がある場合は順序を明示]
```

---

## Phase 4: 実行計画の提示と承認

ユーザーに実行計画を提示：

```
## タスク実行計画

📋 **タスク**: [タスク内容の要約]

### 担当エージェント
| エージェント | 役割 | サブタスク |
|-------------|------|-----------|
| [名前] | [役割] | [担当内容] |

### 実行フロー
1. [ステップ1]
2. [ステップ2]
...

この計画で実行してよろしいですか？
```

AskUserQuestionで承認を取得（選択肢: 「実行する」「修正する」「キャンセル」）

---

## Phase 5: タスク実行

承認後、Taskツールで該当エージェントを起動。

### 5-1. コアエージェントの起動

| エージェント | subagent_type |
|-------------|---------------|
| BOSS | BOSS |
| GENESIS | GENESIS |
| ARCHITECT | ARCHITECT |

### 5-2. 実働部隊の起動

実働部隊エージェントは `subagent_type=general-purpose` で起動し、promptにエージェント定義（AGENT.md）の内容を含める。

```
Task(
  subagent_type: "general-purpose",
  prompt: """
  あなたは[エージェント名]として以下のタスクを実行してください。

  ## エージェント定義
  [AGENT.mdの内容]

  ## タスク
  [タスク内容]

  ## 期待する成果物
  [成果物の説明]
  """
)
```

### 5-3. 複数エージェントの連携

サブタスクが複数ある場合：
- 依存関係がないタスクは並列実行（複数のTaskを同時発行）
- 依存関係があるタスクは順次実行（前のタスク完了を待って次を実行）

---

## Phase 6: 結果の集約と報告

全サブタスクの完了後、結果を集約：

```
## タスク完了報告

📋 **タスク**: [タスク内容]
✅ **ステータス**: 完了

### 実行結果

#### [サブタスク1] - [担当エージェント]
[結果の要約]

#### [サブタスク2] - [担当エージェント]
[結果の要約]

### 成果物
- [成果物1の場所/内容]
- [成果物2の場所/内容]

### 次のステップ（提案）
- [提案1]
- [提案2]
```

---

## エラーハンドリング

### エージェントが見つからない場合
```
⚠️ このタスクに適したエージェントが見つかりませんでした。

提案:
1. GENESISに新しいエージェントを作成してもらう
2. 既存のエージェントにタスクを割り当てる（最も近い: [エージェント名]）
3. タスク内容を変更する

どうしますか？
```

### タスクが曖昧な場合
```
⚠️ タスク内容をもう少し具体的に教えてください。

確認したい点:
- [不明点1]
- [不明点2]
```
