## 概要

新規プロンプトを設計・作成するためのメタプロンプト（統合版）。
構造化されたテンプレート形式と、実践的なシンプルさを両立。

**用途**: エージェントやユーザー向けの高品質なプロンプトを生成

---

## 入力変数

| 変数名 | 説明 | 必須 | デフォルト値 |
|--------|------|------|-------------|
| `{{実現したいこと}}` | プロンプトの目的・達成したいタスク | 必須 | - |
| `{{理想とする出力例}}` | 期待する出力形式・内容の具体例 | 必須 | - |
| `{{使用エージェント}}` | 使用するエージェント名 | 任意 | 汎用 |
| `{{入力要件}}` | 入力として必要なもの（任意で詳細指定） | 任意 | - |
| `{{追加の制約・指示}}` | 特別な制約や追加の指示 | 任意 | - |

---

## プロンプト本文

```
# 前提条件:
- タイトル: 特化型プロンプトを作成する
- 依頼者条件: 特定の目的に合わせた効果的なプロンプトを必要とする人
- 制作者条件: ユーザーの要求を理解し、構造化されたプロンプトを作成できるプロンプトエンジニア
- 目的と目標: ユーザーの「実現したいこと」に基づいて、「理想とする出力例」に従った高品質で再現性の高いプロンプトを生成する
- 評価基準: 生成されたプロンプトがユーザーの意図を正確に反映し、指定されたフォーマットに準拠し、同じ入力で同じ結果が得られること
- 実施要件:
  1. ユーザーから「実現したいこと」と「理想とする出力例」を入力として受け取る
  2. 「実現したいこと」を詳細に分析し、プロンプトに必要な要素を抽出する
  3. 抽出した要素と「理想とする出力例」の構造を適合させる
  4. プロンプト構造（メタ情報、前提条件、変数設定、実行内容、追加指示、フォーマット例、注意事項）に従ってプロンプトを作成する
  5. 作成されたプロンプトが「実現したいこと」と「理想とする出力例」の両方の要件を満たしているか確認する
  6. 最終的なプロンプトを整形して出力する

# 変数設定:
- 実現したいこと = {{実現したいこと}}
- 理想とする出力例 = {{理想とする出力例}}
- 使用エージェント = {{使用エージェント}}
- 入力要件 = {{入力要件}}
- 追加の制約・指示 = {{追加の制約・指示}}

# 実行内容:
{{実現したいこと}}を実行するプロンプトを、以下の「出力フォーマット」に従って作成してください。
{{理想とする出力例}}に近づけることを最終ゴールとします。

# プロンプト設計のルール:

## 1. プロンプトの構成要素

### メタ情報（必須）
- ID（一意の識別子、例: x-post-001）
- バージョン（例: 1.0.0）
- カテゴリ（例: コンテンツ生成、分析、変換）
- 作成日・最終更新日（YYYY-MM-DD形式）
- 作成者
- 使用エージェント

### 概要（必須）
- このプロンプトが何をするか（1-2文で簡潔に）
- 用途を明示

### 前提条件（推奨）
- タイトル
- 依頼者条件・制作者条件
- 目的と目標
- 評価基準
- 実施要件（ステップ形式）

### 入力変数（必須）
- 変数名（{{変数名}}形式）
- 説明
- 必須/任意
- デフォルト値

### プロンプト本文（必須）
- 役割の指定（「あなたは〇〇として行動してください」）
- コンテキストの説明
- 実行ルール（構成、トーン、制約条件、品質基準）
- 出力形式の指定

### 使用例/フォーマット例（必須）
- 入力例
- 出力例
- 構造イメージ

### 注意事項（推奨）
- 使用上の注意点
- 避けるべきこと

### 変更履歴（必須）
- バージョン管理

## 2. 良いプロンプトの特徴

- **明確**: 曖昧さがない
- **構造化**: 論理的に整理されている
- **再現性**: 同じ入力で同じ結果が得られる
- **柔軟性**: 変数で調整可能
- **具体的**: 出力形式が明確
- **実践的**: 実際に使える形式

## 3. 避けるべきこと

- ❌ 曖昧な指示（「適切に」「良い感じに」など）
- ❌ 長すぎる説明（簡潔さを重視）
- ❌ 矛盾した要求（「短く」と「詳細に」の両立など）
- ❌ 出力形式の未指定
- ❌ 例示なし（使用例は必須）
- ❌ 変数の未定義（{{変数}}を使う場合は必ず定義）

# 追加指示:
- プロンプト構造は「出力フォーマット」に従ってください
- {{理想とする出力例}}の構造を分析し、それを再現できる形にしてください
- 変数名は分かりやすく、{{変数名}}形式で記載
- 出力形式は具体的に指定（箇条書き、マークダウン、JSON、テキストなど）
- 使用例は必ず含める（入力例と出力例の両方）
- 「no yapping」 - 余計な前置きや説明は不要、プロンプトだけを出力
- {{追加の制約・指示}}がある場合は、それを「追加指示」セクションに反映してください

## 統合の経緯

このプロンプトは、以下の2つのプロンプトの良いところを統合したものです:

### 統合元1: prompt-creator.md
**採用した要素**:
- メタ情報テーブル（ID、バージョン、カテゴリ、作成日など）
- 入力変数テーブル（必須/任意、デフォルト値）
- 使用例・注意事項・変更履歴
- 構造化されたテンプレート
- 良いプロンプトの原則

### 統合元2: prompt-creation-template.md
**採用した要素**:
- 前提条件（タイトル、依頼者/制作者条件、目的と目標、評価基準、実施要件）
- 変数設定（シンプルな形式）
- 実行内容（簡潔で明確）
- 追加指示（柔軟な補足事項）
- フォーマット例（具体的な出力イメージ）
- 「no yapping」などの実践的な指示

### 統合のメリット
- **構造的な網羅性**: prompt-creator.mdのメタ情報、テーブル形式、変更履歴
- **実践的なシンプルさ**: prompt-creation-template.mdの前提条件、実行内容、追加指示
- **両立**: 丁寧で網羅的でありながら、実践的で使いやすい
