## 概要

既存のプロンプトをClaude Codeスキル（SKILL.md）に変換する、または新規スキルをゼロから設計・作成するためのメタプロンプト。

**用途**: プロンプトのスキル化、新規スキル作成、スキル構造の最適化

---

## 入力変数

| 変数名 | 説明 | 必須 | デフォルト値 |
|--------|------|------|-------------|
| `{{モード}}` | `変換`（既存プロンプト→スキル）または `新規`（ゼロから作成） | 必須 | - |
| `{{元プロンプト}}` | 変換元のプロンプト（変換モード時） | 条件付き必須 | - |
| `{{実現したいこと}}` | スキルの目的・達成したいタスク（新規モード時） | 条件付き必須 | - |
| `{{理想とする出力例}}` | 期待する出力形式・内容の具体例 | 任意 | - |
| `{{使用エージェント}}` | このスキルを使用するエージェント名 | 任意 | 汎用 |
| `{{スキル名}}` | スキルのディレクトリ名（半角英数+ハイフン） | 任意 | 自動生成 |

## プロンプト本文

```
# 前提条件:
- タイトル: Claude Codeスキル（SKILL.md）を作成する
- 依頼者条件: プロンプトをスキル化したい、または新しいスキルを作りたい人
- 制作者条件: Claude Codeのスキル構造を熟知し、高品質なSKILL.mdを設計できるプロンプトエンジニア
- 目的と目標: 再現性が高く、Claude Codeで即座に使えるスキルファイルを生成する
- 評価基準:
  1. SKILL.mdのフロントマター（name, description）が正しく設定されている
  2. スキルの説明文がCLAUDE.mdのスキル一覧に追加できる形になっている
  3. プロンプト本文が明確で再現性が高い
  4. 入力変数が適切に定義されている
  5. 使用例が含まれている

# 変数設定:
- モード = {{モード}}
- 元プロンプト = {{元プロンプト}}
- 実現したいこと = {{実現したいこと}}
- 理想とする出力例 = {{理想とする出力例}}
- 使用エージェント = {{使用エージェント}}
- スキル名 = {{スキル名}}

# 実行内容:

## モード別処理

### 変換モード（既存プロンプト → スキル）
1. {{元プロンプト}}を分析し、以下を抽出する:
   - 目的（何を達成するプロンプトか）
   - 入力変数（何を受け取るか）
   - 処理ロジック（何をどう処理するか）
   - 出力形式（何をどう出力するか）
   - 制約条件（守るべきルール）
2. 抽出した要素をSKILL.md形式に再構成する
3. 元プロンプトの意図を損なわず、スキルとしての使いやすさを向上させる

### 新規モード（ゼロから作成）
1. {{実現したいこと}}を分析し、必要な要素を洗い出す
2. {{理想とする出力例}}があればそれに合わせて設計する
3. SKILL.md形式でスキルを設計する

## スキル設計のルール

### SKILL.mdの必須構造
1. **フロントマター**（YAML形式）
   - `name`: スキル名（半角英数+ハイフン、SKILL.mdのディレクトリ名と一致）
   - `description`: 1行で何をするスキルか説明（日本語OK）

2. **プロンプト本文**
   - 入力変数の定義（`{変数名}` 形式）
   - 前提条件（タスク、依頼者条件、制作者条件）
   - 実行内容（具体的な処理手順）
   - 追加指示（制約条件、品質基準）
   - 出力形式（期待する出力のフォーマット）

3. **Knowledge参照**（該当する場合）
   - 参照すべき知識ファイルのパスを明示

### フロントマターの書き方
- `name` はディレクトリ名と完全一致させる
- `description` はCLAUDE.mdのスキル一覧にそのまま転記できる文にする
- 余計なフロントマター項目は追加しない（name と description のみ）

### 良いスキルの特徴
- 1スキル1責務（1つの明確な目的）
- 入力変数が明確に定義されている
- ステップバイステップで処理が進む
- 出力形式が具体的に指定されている
- 再現性が高い（同じ入力で同じ品質の結果が得られる）

### 避けるべきこと
- フロントマターにname, description以外の項目を入れる
- 入力変数の定義が曖昧
- 処理手順が不明確
- 出力形式が未指定
- 1つのスキルに複数の責務を詰め込む

# 追加指示:
- 「no yapping」 - 余計な前置きや説明は不要、SKILL.mdの内容だけを出力
- フロントマターは必ず `---` で囲む
- プロンプト本文はマークダウン形式で構造化する
- {{使用エージェント}}が指定されている場合、そのエージェントの特性に合わせて最適化する
- 変換モードの場合、元プロンプトの品質が低い箇所は改善した上でスキル化する

# 出力形式:

以下の形式でSKILL.mdの完全な内容を出力してください。

"""
---
name: [スキル名]
description: [1行の説明]
---

# [スキルのタイトル（日本語）]

## 入力:
- [変数1] = {変数1}
- [変数2] = {変数2}

## 前提条件:
- タスク: [タスク説明]
- 依頼者条件: [依頼者の条件]
- 制作者条件: [制作者の条件]

## 実行内容:
[具体的な処理手順をステップバイステップで]

## 追加指示:
- [制約条件1]
- [制約条件2]

## 出力形式:
[期待する出力のフォーマット]
"""

# 補足:
- 出力後、以下の情報も添える:
  1. スキルのディレクトリ名（`.claude/skills/[name]/SKILL.md` として保存）
  2. CLAUDE.mdに追加するスキル一覧エントリ（`| スキル名 | /コマンド名 | 説明 |` 形式）
  3. エージェントのskillsフロントマターに追加する名前
```

## 使用例

### 入力例1（変換モード）

```
モード: 変換

元プロンプト:
あなたはSEO専門家です。
以下のキーワードについて、検索意図を分析し、記事の見出し構成を提案してください。
- キーワード: {{keyword}}
- ターゲット読者: {{target}}
出力は見出し構成（H2, H3）のマークダウン形式で。

使用エージェント: NOTE_CREATOR
スキル名: seo-outline
```

### 出力例1

```markdown
---
name: seo-outline
description: キーワードの検索意図を分析し、SEO最適化された記事見出し構成を提案する
---

# SEO見出し構成設計

## 入力:
- キーワード = {キーワード}
- ターゲット読者 = {ターゲット読者}

## 前提条件:
- タスク: {キーワード}の検索意図を分析し、SEO最適化された見出し構成を設計する
- 依頼者条件: SEOを意識した記事構成が必要なライター
- 制作者条件: 検索意図分析とSEOライティングに精通した専門家

## 実行内容:
1. {キーワード}の検索意図を分析する（情報収集型/取引型/案内型）
2. 競合が取り上げている主要トピックを推定する
3. {ターゲット読者}のニーズに合わせた見出し構成を設計する
4. H2（大見出し）とH3（中見出し）の階層構造で提案する

## 追加指示:
- H2は5〜8個を目安にする
- 各H2にはH3を2〜4個含める
- キーワードをH2に自然に含める
- 読者の疑問に答える構成にする

## 出力形式:
マークダウンの見出し構成（H2, H3の階層構造）
```

**保存先**: `.claude/skills/seo-outline/SKILL.md`
**CLAUDE.mdエントリ**: `| SEO見出し構成 | /seo-outline | キーワードからSEO最適化見出し構成を提案 |`
**skillsフロントマター**: `- seo-outline`

### 入力例2（新規モード）

```
モード: 新規

実現したいこと: YouTubeのサムネイルテキストを5案提案するスキル

理想とする出力例:
案1: 【衝撃】ChatGPTが仕事を奪う日
案2: 99%の人が知らないAI活用術
案3: もう戻れない。AI時代の働き方
案4: プロが教えるChatGPT裏ワザ5選
案5: これ知らないと損するAIツール

使用エージェント: X_POST_CREATOR
スキル名: yt-thumbnail-text
```

## 注意事項

- フロントマターは `name` と `description` のみ。余計な項目を入れない
- `name` はディレクトリ名と完全一致させる（半角英数+ハイフン）
- プロンプト本文内の変数は `{変数名}` 形式（二重波括弧ではなく一重）
- 変換モードでは元プロンプトの意図を壊さないよう注意する
- 新規モードでは「1スキル1責務」を厳守する
