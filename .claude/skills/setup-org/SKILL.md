# 組織セットアップスキル

ユーザーがセットアップを依頼しました。テンプレートとして用意されている組織ファイル群を、ユーザー固有の情報（ジャンル・文体・商品・ペルソナ等）に合わせて改変・最適化します。以下の Phase を順番に実行してください。

---

## Phase 0: 事前読み込み

以下を読み込んでから開始すること：

1. `ai-agent-organization/system/config.yaml` — 現在の設定値
2. `ai-agent-organization/shared/knowledge/x-note-brand-strategy.md` — 既存の戦略情報（あれば）

---

## Phase 0.5: オリエンテーション

ユーザーに以下をテキスト出力で伝える：

```
これから組織セットアップを始めます。

**やること:**
1. ヒアリング（質問リストに回答してもらいます・全3回）
2. サンプルがあれば文体解析
3. 40以上のファイルを自動生成・更新

**所要時間の目安:** 15〜30分（回答のペースによります）
**準備しておくと良いもの:**
- 過去に書いたnote記事やX投稿のサンプル（あれば）
```

---

## Phase 1: 成果物の確認（inbox チェック）

**最初に必ずこの確認をすること。**

以下をテキスト出力でユーザーに伝える：

```
これまでに書いたnote記事やX投稿のサンプルはありますか？

あれば `ai-agent-organization/inbox/` フォルダに入れてください。
サンプルをもとに文体・口調・テーマを自動抽出し、
エージェントをあなたの書き方に最適化します。

（なくてもセットアップは進められます。「なし」でも大丈夫です）
```

→ **「あり」の場合**: ユーザーがinboxに配置したら `ai-agent-organization/inbox/` を読み込み、Phase 2 の後に Phase 3a（サンプル解析）を実行する
→ **「なし」の場合**: Phase 2 に進む

---

## Phase 2: ヒアリング（3ラウンド構成）

**AskUserQuestionは使わない。** 質問リストをテキスト出力で一括提示し、ユーザーにまとめて回答してもらう方式。
「未定」「後で」と答えた項目は `[未定]` プレースホルダーで残す。

---

### ラウンド0: 発信フェーズ判定

まず以下の1問だけをテキスト出力で提示する：

```
あなたの現在の発信フェーズを教えてください:
A: これから始める（まだ発信していない）
B: 始めたばかり（数ヶ月以内）
C: 既に運用中（半年以上、商品もある程度決まっている）
```

→ この回答で以降のラウンド2・3の質問セットを切り替える

---

### ラウンド1: 基本情報 + 文体 + 権威性

以下の質問リストをテキスト出力で一括提示する：

```
以下の質問にまとめて回答してください:

【アカウント基本情報】
1. Xアカウント名 / ハンドルネーム（例: @tanaka_ai）
2. メインテーマ・発信ジャンル（例: 副業・マーケティング・料理・育児など）
3. 発信開始時期（例: 2024年4月〜）
4. 現在使っている発信チャネル（例: X / note / YouTube / ブログ / メルマガ）

【文体・トーン】
5. 一人称（例: 僕 / 私 / 俺）
6. 読者との距離感はどれに近いですか？
   A: 友達に話すような気軽さ
   B: 先輩が教えるような安心感
   C: 専門家として淡々と解説
   D: 熱血で引っ張る
   E: その他（自由に）
7. 特徴的な口癖・よく使うフレーズ（あれば）
8. 絶対に使いたくない表現（あれば）
9. 改行スタイルの好み（1文1行 / 2-3行で段落 / 特にこだわりなし）

【権威性・実績】
10. 肩書き・経歴（読者に信頼される根拠になるもの）
11. 数値で示せる実績（フォロワー数・収益・指導人数など。まだない場合は今の取り組みでOK）
12. 読者が共感できるエピソード（失敗談・転機など。あれば）
```

#### フェーズ別の補足
- **フェーズA/Bの場合**: 質問10〜12に「実績がまだない場合は、今取り組んでいること・目指していることでもOK」と補足を追加して提示する

#### 変数マッピング
| 質問番号 | 反映先変数 |
|---------|-----------|
| 1 | `[発信者名]` |
| 2 | `{ジャンル}` |
| 6 | キャラクター設定 |
| 10 | 肩書き |
| 11 | `{発信者の強み②}`（数値実績） |
| 12 | 自己開示エピソード |

### ラウンド2: ターゲット読者・ペルソナ設計（AI提案型）

ラウンド1の回答（特にジャンル）を踏まえ、動的に質問・提案を行う。

#### Step 2a: 自由記述で読者像を聞く

ラウンド1のジャンルに合わせた具体例を含めて、以下をテキスト出力する：

```
あなたのコンテンツを読みに来る人は、どんな人ですか？
思いつくままに書いてください（箇条書きでも文章でもOK）

例（{ジャンル}の場合）:
- {ジャンルに合わせた読者例1}
- {ジャンルに合わせた読者例2}
- {ジャンルに合わせた読者例3}
```

#### Step 2b: AIがセグメント候補を提案

ユーザーの回答 + ラウンド1のジャンル情報から、3〜5個の関心セグメントを提案する。

提案フォーマット（テキスト出力）:
```
あなたの読者を以下のセグメントに分けてみました:

1. 【{セグメント名}タイプ】{1行説明}
   - 核心動機: {なぜこのテーマを追いかけるか}
   - ペイン: {今まさに困っていること}
2. 【{セグメント名}タイプ】{1行説明}
   - 核心動機: ...
   - ペイン: ...
（3〜5個）

→ この分類でOKですか？修正・追加・削除があれば教えてください
```

#### Step 2c: ペルソナ名の確定

セグメント確定後、各セグメントにペルソナ名を提案する（テキスト出力）:
```
各セグメントにペルソナ名をつけました:
1. {ペルソナ名}（{年代}{職業}、{核心動機の要約}）
2. {ペルソナ名}（{年代}{職業}、{核心動機の要約}）
（セグメント数分）

→ OKですか？変更したい名前があれば教えてください
```

#### Step 2d: 各セグメントの詳細（商材関心）をAIが草案

ペルソナ名確定後、各セグメントの商材への関心をAIが草案提示 → ユーザーが修正・承認:
```
各ペルソナの商材への関心度を整理しました:

| ペルソナ | 最も関心が高い商材タイプ | 購買動機 |
|---------|----------------------|---------|
| {ペルソナ名1} | {商材タイプ} | {動機} |
| {ペルソナ名2} | {商材タイプ} | {動機} |

→ 修正点はありますか？
```

### ラウンド3: 商品・マネタイズ + コンテンツテーマ

フェーズによって質問セットを切り替える。

#### フェーズA用（簡略版）

```
以下の質問にまとめて回答してください:

【商品・マネタイズの方向性】
1. 将来的にどんな形で収益化したいですか？（方向性だけでOK）
   例: 有料記事 / オンライン講座 / コミュニティ / コンサル / まだ考えていない
2. 販売プラットフォームの候補（あれば）
   例: note / Zenn / Brain / 自前ブログ / 未定

※ 商品の詳細設計はセットアップ後にGUNSHI（官兵衛）と壁打ちで詰められます。
  今は方向性だけでOKです。

【コンテンツテーマ】
3. 主要テーマ（3-5個）（例: AI活用・副業収益化・時間短縮術）
4. 絶対に扱わないテーマ（例: 政治・宗教・特定個人の批判）
5. あなたならではの強み・切り口（例: 再現性の高さ・初心者目線・業界経験）
```

フェーズAの場合、商品関連項目は `[未定]` で残す。

#### フェーズB/C用（フル版）

```
以下の質問にまとめて回答してください:

【商品・マネタイズ設計】
1. 無料コンテンツの役割（例: 信頼構築・認知拡大・リスト獲得）
2. 有料商品①（名称・価格）（例: 有料記事 1,000-3,000円 / テンプレ販売 / 動画講座）
3. 有料商品②（名称・価格）（例: オンラインコミュニティ 月額3,000-5,000円）
4. 販売プラットフォーム（例: note / Zenn / Brain / 自前ブログ / 未定）
5. ファネル設計（例: 無料X投稿 → 無料記事 → 有料商品 → 継続商品）
6. CTAの置き方（例: 記事末尾に自然な形で案内）

【コンテンツテーマ】
7. 主要テーマ（3-5個）（例: AI活用・副業収益化・時間短縮術）
8. 絶対に扱わないテーマ（例: 政治・宗教・特定個人の批判）
9. 競合との差別化ポイント（例: 再現性の高さ・初心者目線・具体的な手順）
10. あなただけの強み（例: 業界経験・独自の失敗体験・特殊なスキル）
```

#### 変数マッピング
| 質問（フル版番号） | 反映先変数 |
|------------------|-----------|
| 2 | `{フロント商品}` |
| 3 | `{ミドル商品}` `{サロン名}` |
| 7 | `{テーマタグ①②}` |
| 9 | `{発信者の強み③}` |
| 10 | `{発信者の強み①}`（専門性の根拠） |

## Phase 3a: サンプル解析（inbox に成果物がある場合のみ）

`ai-agent-organization/inbox/` 内の全ファイルを読み込み、以下を抽出する：

### 抽出項目

**文体プロファイリング（→ writing-style-profile.md に反映）**
- 一人称の実際の使い方
- 語尾パターンと出現頻度（→ sentence-ending-patterns.md に反映）
- 特徴的な接続表現・言い回し
- 段落の長さ・改行リズム
- 感情表現・自己開示のパターン
- 読者への話しかけ方
- NG表現（AIっぽさのある表現が含まれていた場合は記録）

**コンテンツ傾向（→ analysis-methods.md・x-post-best-practices.md に反映）**
- 実際に使われているテーマ・キーワード → `{テーマタグ}` の精緻化
- 高頻度で使うフック・リード文のパターン
- CTAの置き方・表現

**note記事サンプルがある場合**
- `organization/operations/note-creator-NOTE_CREATOR/knowledge/examples/` に配置
- 見出し構成のクセ・文章構造を writing-style-profile.md に追記

**X投稿サンプルがある場合**
- `organization/operations/x-post-creator-X_POST_CREATOR/knowledge/examples/` に配置
- フック・改行・絵文字使用パターンを writing-style-profile.md に追記

### 解析結果のフィードバック

サンプル解析完了後、以下の形式でユーザーにテキスト出力で結果を提示する：

```
サンプルから以下の特徴を抽出しました:

【文体の特徴】
- 一人称: 「{実際の一人称}」
- 語尾の傾向: 「{頻出語尾1}」「{頻出語尾2}」が多い
- 段落: {短め/長め}（{特徴の説明}）
- 特徴的な表現: 「{特徴的フレーズ1}」「{特徴的フレーズ2}」

【コンテンツの傾向】
- よく扱うテーマ: {テーマ1}、{テーマ2}
- フックのパターン: {パターンの説明}
- CTAスタイル: {CTAの特徴}

ヒアリング回答との差異:
- {差異がある場合、具体的に列挙}
  → どちらを採用しますか？

この分析結果で合っていますか？修正点はありますか？
```

→ ユーザーの確認・修正を経てから Phase 4 へ進む

## Phase 4: 全ファイル生成・更新

ヒアリング（＋サンプル解析）の結果をもとに、以下のファイルを更新する。

### 【必須ルール】新規作成より既存ファイルの改変を優先する

**エージェント定義・スキル・ナレッジ、すべてのファイルに適用する。**

各ファイルを扱う前に必ず以下を確認すること：

1. **既存ファイルがある場合 → 内容を改変・上書きする**（前ユーザーの情報を残さない）
2. **既存ファイルがない場合 → 新規作成する**

下表で `【既存があれば改変・なければ新規作成】` と記載されているファイルも、すでに存在する場合は改変で対応する。
**新規ファイルの作成は、ファイルが存在しない場合のみ**行うこと。

#### ファイル種別ごとの判断基準

同じ役割・目的の判断は柔軟に考えて行うこと。

| ファイル種別 | 同じ役割・目的のものが既存にある場合 | 全く新しい役割が必要な場合 |
|------------|--------------------------------|------------------------|
| エージェント定義（`.claude/agents/*.md`） | 既存ファイルの内容を書き換える | 新規作成してよい |
| スキル（`.claude/skills/*/SKILL.md`） | 既存スキルの内容を書き換える | 新規作成してよい |
| ナレッジ（`knowledge/**/*.md`） | 既存ファイルの内容を書き換える | 新規作成してよい |

### Phase 4 実行時の進捗出力ルール

各グループの処理完了ごとに以下をテキスト出力する：

```
✅ グループX完了（{グループ名}: {N}ファイル更新）
  → {ファイル名1} / {ファイル名2}

⏳ グループY処理中（{グループ名}）...
```

特に重要なファイル（以下3つ）は生成後にサマリーを表示:
- **x-note-brand-strategy.md** → 主要セクション見出しを表示
- **writing-style-profile.md** → 文体ルールの要点を表示
- **ペルソナファイル** → 各ペルソナの1行サマリーを表示

### グループE: 記事レビューエージェント（ARTICLE_REVIEWER）

ペルソナ定義はknowledgeの個別ファイルに分離されています。
ラウンド2で特定した関心セグメントに対応するペルソナファイルを用意します。

#### ペルソナファイルの扱いルール

既存のスケルトンファイルが4つあります：
- `persona-01-problem-solver.md`（即課題解決型）
- `persona-02-skill-investor.md`（スキル投資・成長型）
- `persona-03-community-seeker.md`（つながり・共感型）
- `persona-04-roi-optimizer.md`（ROI最適化型）

**ラウンド2のセグメントへの対応方針（優先順位順）：**

1. **スケルトンの数（4個）以内のセグメント数の場合 → スケルトンを直接改変する**
   - 各スケルトンファイルの内容をラウンド2のセグメントに合わせて書き換える
   - 使わないスケルトンファイルは内容を空にするのではなく、そのまま残す

2. **セグメント数がスケルトン数を超える場合 → 超えた分だけ新規作成する**
   - `persona-NN-[セグメント識別名].md` の形式で追加分のみ新規作成

### エージェント定義ファイル（`.claude/agents/*.md`）について

`.claude/agents/` 配下のエージェント定義ファイル自体（`NOTE_CREATOR.md` 等）は**基本的に編集不要**。
各エージェントはタスク実行時に `knowledge/` ファイルを動的に参照するため、グループC〜H の knowledge ファイルを更新すれば自動的に反映される。

ただし以下の場合はエージェント定義ファイルも編集が必要：

| 変更ケース | 編集対象ファイル |
|-----------|----------------|
| エージェントの人格・口調を根本的に変えたい | `.claude/agents/[AGENT_NAME].md` の「人格・口調」セクション |
| エージェントが参照するスキルを追加・変更したい | `.claude/agents/[AGENT_NAME].md` の `skills:` フロントマター |
| プラットフォームや役割が変わる場合（例: X投稿 → Threads投稿） | **既存の `.claude/agents/[AGENT_NAME].md` を直接編集**して内容・名称を書き換える。新規ファイル作成＋旧ファイル削除はしない |

### 【必須ルール】同じ役割のエージェントは既存ファイルを改変する

役割が同じ（または近い）エージェントを変更する場合、**新規ファイルを作らず既存ファイルを編集する**こと。

- NG: `X_POST_CREATOR.md` を残したまま `THREADS_CREATOR.md` を新規作成する
- OK: `X_POST_CREATOR.md` の中身（名称・説明・スキル参照・人格設定）を書き換えて Threads 投稿エージェントとして機能させる

新規エージェントファイルの作成が許されるのは、**既存エージェントでは担えない全く新しい役割**が必要な場合のみ。
