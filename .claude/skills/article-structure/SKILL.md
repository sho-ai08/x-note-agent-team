# 記事構成作成プロンプト

## 概要
読者を惹きつけるコンテンツの見出し構成を設計するプロンプト。
NOTE_CREATORが記事本文を執筆する前に、全体の構成を決定する際に使用。

---

# 入力:
- コンテンツ案 = {コンテンツ案}

# Knowledge参照:
- `ai-agent-organization/shared/knowledge/noteの心得.md`: note記事作成の心得
- `ai-agent-organization/organization/operations/note-creator-NOTE_CREATOR/knowledge/examples/`: 実際のnote記事例
- `ai-agent-organization/organization/operations/note-creator-NOTE_CREATOR/knowledge/knowhow/writing-style-profile.md`: 著者の文体プロファイル
- `ai-agent-organization/organization/operations/note-creator-NOTE_CREATOR/knowledge/knowhow/article-creation-process.md`: 記事作成プロセス

# 前提条件:
- タスク: コンテンツ案をもとにコンセプトを設計し、コンテンツの見出し構成を最適化する
- 依頼者条件: 読者を魅了するコンテンツ構成を作成したい個人
- 制作者条件: 読者を惹きつけるコンテンツ構成の設計ができるライター
- 目的と目標: コンテンツ案をもとにコンセプトを設計し、読者が満足するコンテンツの見出し構成を目指す

# 変数設定:
- コンテンツ案 = {コンテンツ案}

# 実行内容:
入力された{コンテンツ案}から売れるコンセプトを設計し、
ターゲットとなる読者を想定して、コンセプトから悩みの順序を明確にしてください。
そして、悩みの順序から悩みをすべて解決できる大枠の章構成でh2見出しの見出し構成を作成してください。

なお、{ステップ形式}に従って、ステップバイステップで実行してください。

# step形式:
それぞれのstepが終われば、「次に進んでも良いですか？」と強制的に質問してください。

## step1 コンセプト設計:
入力された{コンテンツ案}から売れるコンセプトを設計してください。
考えられるターゲット属性やコンテンツ案の独自性、コンテンツのスタートとゴールを定義した上で、感動分岐点を大きく超えるように詳細に設計してください。

### step1の追加指示:
- NOTE_CREATORの `knowhow/strategy-brief.md` と整合するコンセプトであること
- 最後に「改善点があれば教えてください。何もなければ"次"と入力してください」と質問
- ユーザから改善点が提示されたら、改善点を修正したコンセプト案を再度設計
- "次"と入力されるまで改善をループすること

## step2 悩みの順序設計:
[step1]で決定したコンセプトをもとに、ターゲットとなる読者を想定して、スタートからゴールの間に抱える悩み(インサイト)の順序を明確にしてください。
なお、各悩みを解決するための記事を書くことを想定して逆算すること。

### step2の出力フォーマット:
```
スタート：[読者の初期状態]
↓
悩み①：[悩みの内容]
悩みを解決するために必要なこと：[解決に必要な要素]
見出し案：[仮の見出し]
↓
悩み②：[悩みの内容]
悩みを解決するために必要なこと：[解決に必要な要素]
見出し案：[仮の見出し]
↓
...
↓
ゴール：[読者の理想状態]
```

## step3 見出し構成作成:
[step2]で決定したターゲットの悩みの順序をもとに、悩みをすべて解決できる大枠の章構成の流れで見出し構成(h2見出し)を作成してコードブロック形式で出力してください。

### step3の追加指示:
- [フォーマット例1]と[フォーマット例2]の2パターンを出力し、ユーザに選択させること
- h2見出しの数は内容や悩みの数に応じて過不足ないように最適化
- 全体の流れが論理的で読者を最後まで引き付ける見出し構成にすること
- 各見出しは読者の常識(メンタルモデル)を塗り替えるような興味を惹く書き方をする
- 少し砕けた口調で書くこと
- `ai-agent-organization/organization/operations/note-creator-NOTE_CREATOR/knowledge/knowhow/writing-style-profile.md` の口調を参照

### フォーマット例1（文章型）:
読者の悩みにドンピシャで刺さる見出しが書ける
```
## 問題点と理想の未来をサンドイッチせよ
## 問題点の指摘をして「顧客の現在地点」を定義する
## 理想の未来を見せて「顧客の理想郷」を定義する
## 「差」を拡げるほど勝ちだし価値である
```

### フォーマット例2（記事型）:
少しかっちりとした記事構成を作れる
```
## 3日間で売れる仕組みを作る禁断の方法
## 「顧客の現在地点」を勝手に決めてしまえる問題点の教育
## ビジネスの3大要素｜集客・教育・販売
```

## step4 必要リソースの列挙:
[step3]で決定した見出し構成から、記事を完成させるために必要な要素を列挙してください。
知識や一次情報、事例集など棚卸しすべきリソースを列挙すること。

### step4の出力フォーマット:
```
【記事作成に必要なリソース】
- [必要なリソース①]
- [必要なリソース②]
...
```

## step5 次のアクション:
「次」と入力されたら、次のアクションを提案してください。

```
【次のアクション】
1. 必要リソースの準備・収集
2. `note-creation` スキルで本文を執筆
3. ARTICLE_REVIEWERにレビュー依頼
4. ARTICLE_STYLISTに装飾依頼
```

# 絶対条件:
- 指示の復唱や余計な補足は入れないでください
- 見出し構成の最終確認では、悩みをすべて抜け漏れなく解決できそうか確認すること
- `ai-agent-organization/shared/knowledge/noteの心得.md` を参照して整合性を確保すること
