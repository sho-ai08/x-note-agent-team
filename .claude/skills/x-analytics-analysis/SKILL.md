# X Analytics 分析スキル

## 概要

ユーザーが提供するX(Twitter)のanalytics CSVデータを分析し、投稿パフォーマンスの傾向を可視化して、X運用改善のための具体的なアクションを提案する。

**用途**: X運用を改善するための定量的な根拠づくり

---

- 分析観点 = {分析観点}（任意: 全体/投稿タイプ別/テーマ別、省略時は全観点）

# 実行内容:

## Step 1: データ受領・前処理

CSVを読み込み以下を処理:

**1-1. 投稿タイプの分類**（優先度順に判定）:
- **リプライ投稿**: ポスト本文が`@`で始まる
- **URL共有のみ**: ポスト本文がURLのみ（t.co等のURLのみで他テキストなし）
- **note告知投稿**: 【...】の告知文がある or note/t.coリンクと告知テキストが共存する
- **通常コンテンツ投稿**: 上記以外

**1-2. エンゲージメント率の計算**: `エンゲージメント / インプレッション × 100`（小数第2位まで。インプレ0はスキップ）

**1-3. テーマタグ付け**（通常コンテンツ投稿のみ、主要キーワードから自動判定）:
- `Claude/Claude Code系`: Claude、Claude Code等を含む
- `AIツール系`: Antigravity、Cowork、ChatGPT、Gemini、Manus等を含む
- `思考・哲学・AI論系`: 「価値」「考え」「時代」「学ぶ」「強い」等の抽象論
- `情報発信・副業系`: 発信者、副業、収益、フォロワー等を含む
- `サービス告知系`: LOOP、サロン、講座等を含む
- `その他`: 上記に当てはまらないもの

# 制約条件:
- リプライ投稿・URL共有のみは、全体サマリーと投稿タイプ比較には含めるが、テーマ分析・曜日分析からは除外
- インプレッション数0の投稿は全集計から除外し、その旨を記載
- テーマ分類が不明な投稿は「その他」とする
- 5件未満のカテゴリには「サンプル少（X件）」と明記
- 推測による補完は行わない。データ不足の場合は明記

# 絶対条件:
- 指示の復唱・余計な前置きなし
- 忖度禁止: パフォーマンスが悪い投稿タイプも数値で明確に指摘する
- 根拠主義: 数値に基づかない感覚的な提案は行わない
- テーブルは必ずMarkdown形式で出力
- 確信度を明示しない提案は行わない
